## 一、全栈技术全景图

```
┌─────────────────────────────────────────────────────────┐
│                     你的技术栈                          │
├──────────────┬──────────────────┬───────────────────────┤
│    前端      │      后端        │       客户端          │
├──────────────┼──────────────────┼───────────────────────┤
│ HTML/CSS/JS  │ Node.js/Python   │ 移动端：SwiftUI       │
│ React/Vue    │ Express/Django    │ 桌面端：Electron      │
│ Vite/Webpack │ MySQL/Redis      │ 跨平台：Flutter       │

```

**一句话理解**：
- 前端：你在屏幕上看到的东西
- 后端：处理数据、存数据的服务
- 客户端：在手机/电脑上的App

## 二、后端基础（核心）

### 什么是后端？

```
手机App ──────▶ 后端服务 ──────▶ 数据库
 (客户端)      (你的代码)       (存数据)

举例：你在FitGenius订训练计划
1. App发送请求："给我一个训练计划"
2. 后端收到，调用大模型API，生成计划
3. 存到数据库
4. 返回给App显示
```

### 后端技术一览

| 技术 | 语言 | 特点 | 代表网站 |
|------|------|------|----------|
| **Node.js** | JavaScript | 非阻塞IO，适合高并发 | Netflix、Uber |
| **Python (Django/Flask)** | Python | 语法简单，AI友好 | Instagram、Pinterest |
| **Go (Gin)** | Go | 高性能，并发强 | 字节跳动 |
| **Java (Spring)** | Java | 企业级，稳定 | 银行后台 |

### Node.js 示例（最简单）

```javascript
// 一个最简单的后端
const express = require('express');
const app = express();

// 定义接口
app.get('/hello', (req, res) => {
    res.send('你好！');
});

// 启动服务，监听3000端口
app.listen(3000);
```

### 数据库类型

| 类型 | 代表 | 用途 |
|------|------|------|
| **关系型** | MySQL、PostgreSQL | 结构化数据（用户表、订单表） |
| **非关系型** | MongoDB、Redis | 文档存储、缓存 |
| **向量数据库** | Milvus | AI知识库（RAG用） |

### 部署方式（个人开发者）

| 方式 | 价格 | 适合场景 |
|------|------|----------|
| **云服务器（VPS）** | 30-100元/月 | 需要24小时运行 |
| **云函数（Serverless）** | 免费额度大 | 个人项目、小流量 |
| **BaaS（LeanCloud/Firebase）** | 免费有限 | 不想写后端代码 |

```
接口设计原则：
```

## 四、移动端开发形式（重要）

### 四种开发方式

| 方式 | 原理 | 代表App | 特点 |
|------|------|---------|------|
| **原生开发** | 平台官方语言 | 微信、抖音 | 性能最好 |
| **跨平台框架** | 一套代码多端运行 | 闲鱼(Flutter)、抖音(RN) | 效率高 |
| **小程序** | 运行环境 | 美团外卖小程序 | 不用下载 |
| **H5/Web App** | 浏览器运行 | 移动端网页 | 最灵活 |

### 原生开发

```
iOS：Swift + SwiftUI（Apple官方）
Android：Kotlin + Jetpack Compose（Google官方）

你的FitGenius：SwiftUI → iOS原生开发
```

### 跨平台框架

| 框架 | 语言 | 如何工作 | 代表作 |
|------|------|----------|--------|
| **Flutter** | Dart | 自己画UI（Skia引擎） | 淘宝、闲鱼 |
| **React Native** | JS/TS | 调用原生组件 | 抖音、拼多多 |

**区别**：
- Flutter：自己画，性能好，包体积大
- React Native：调用原生，生态好，有性能损耗

## 六、登录认证方式

### 常见方式

| 方式 | 原理 | 应用 |
|------|------|------|
| **手机号+验证码** | 短信验证 | 国内App主流 |
| **微信登录** | OAuth 2.0 | 第三方授权 |
| **Apple登录** | OAuth 2.0 | iOS必需 |

### 手机号登录流程

```
1. 用户输入手机号 → 获取验证码
2. 后端生成验证码，存Redis（5分钟有效）
3. 调用短信服务商发送（阿里云SMS）
4. 用户输入验证码 → 后端验证
5. 验证成功 → 返回Token
```

### 微信登录（OAuth 2.0）

```
1. 点击微信登录 → 跳转微信授权页
2. 用户授权 → 返回Code（临时凭证）
3. 服务端用Code换Token
4. 用Token获取用户信息（昵称、头像）
5. 生成自己的Token返回App
```

### Token 存储

```
移动端：Keychain（安全，持久）
Web端：LocalStorage
每次请求Header带：Authorization: Bearer <token>
```

## 八、HTTP 协议

### 核心概念

| 概念 | 说明 |
|------|------|
| **HTTP vs HTTPS** | HTTPS = HTTP + TLS，加密更安全 |
| **GET vs POST** | GET获取，POST提交 |
| **状态码** | 2xx成功 / 4xx客户端错误 / 5xx服务端错误 |

### 常用状态码

```
200 OK       成功
201 Created  创建成功
400 Bad Request  请求错误
401 Unauthorized 未授权
404 Not Found    未找到
500 Server Error 服务器错误
```

### 如何保持登录（无状态HTTP）

```
1. Cookie：浏览器存储，自动带在请求里
2. Token：手动放在Header里（移动端常用）
```
