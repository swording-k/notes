# 虚函数详解（从实际场景出发）

## 一、先想一个现实问题

假设你在写一个游戏引擎，有这些角色：

```cpp
class Dog {
public:
    void speak() {
        cout << "汪汪汪" << endl;
    }
};

class Cat {
public:
    void speak() {
        cout << "喵喵喵" << endl;
    }
};

class Bird {
public:
    void speak() {
        cout << "叽叽叽" << endl;
    }
};
```

### 问题来了

游戏里有100种动物，你要写一个函数让它们叫：

```cpp
// 你不能这样写：
void letSpeak(Dog* d) { d->speak(); }
void letSpeak(Cat* c) { c->speak(); }
void letSpeak(Bird* b) { b->speak(); }
// ... 要写100个函数！
```

### 你想要的效果

```cpp
// 只想写一个函数，传入任何动物，它都能叫
void letSpeak(??? animal) {
    animal->speak();  // 自动调用对应的叫声
}
```

---

## 二、解决方案：用基类 + 虚函数

### 2.1 第一步：创建一个"动物"基类

```cpp
class Animal {
public:
    // virtual = 虚函数
    // 这个函数在 Animal 里没有实际意义
    // 它的作用是"定义一个规矩"：所有动物必须会叫
    virtual void speak() {
        // 空实现
    }
};
```

### 2.2 第二步：具体动物继承并实现

```cpp
class Dog : public Animal {
public:
    void speak() override {  // override 可加可不加
        cout << "汪汪汪" << endl;
    }
};

class Cat : public Animal {
public:
    void speak() override {
        cout << "喵喵喵" << endl;
    }
};
```

### 2.3 第三步：用基类指针调用

```cpp
void letSpeak(Animal* animal) {
    animal->speak();  // 这一句代码，自动调用正确的版本！
}

int main() {
    Dog d;
    Cat c;

    letSpeak(&d);  // 输出：汪汪汪
    letSpeak(&c);  // 输出：喵喵喵

    return 0;
}
```

## 三、为什么父类要写虚函数？

**因为父类在定义一个"规矩/接口"：**

```
Animal（动物基类）说：
    "我不管你是什么动物，但作为动物，你必须会 speak()"

Dog（狗）说：
    "好，我 speak() 的时候叫汪汪汪"

Cat（猫）说：
    "好，我 speak() 的时候叫喵喵喵"
```

### 类比理解

```
Java/Python 的 interface（接口）：
    public interface Animal {
        void speak();  // 只有声明，没有实现
    }

C++ 的 virtual 函数：
    相当于 C++ 版的"接口"
    基类定义了规矩（虚函数），子类来实现（override）
```

## 四、纯虚函数（更彻底的接口）

### 4.1 普通的虚函数

```cpp
class Animal {
public:
    virtual void speak() {
        // 可以有默认实现
    }
};
```

### 4.2 纯虚函数 = 强制子类必须实现

```cpp
class Animal {
public:
    // = 0 表示"这个函数没有默认实现"
    // 子类必须自己实现，否则子类也不能创建对象
    virtual void speak() = 0;
};
```

### 4.3 图解对比

```
Animal（基类）
├── virtual void speak() { }     // 普通虚函数，有默认实现
└── virtual void eat() = 0;       // 纯虚函数，没有实现，子类必须重写

Dog（子类）
├── void speak() { }               // 继承了默认实现，可以不写
└── void eat() { }                 // 必须实现，否则编译报错
```

### 4.4 纯虚函数的意义

```cpp
class Animal {
public:
    virtual void speak() = 0;  // 纯虚函数
    virtual void eat() = 0;    // 纯虚函数
};

// Animal 是抽象类，不能创建对象
// Animal *p = new Animal();  // 编译报错！

class Dog : public Animal {
public:
    void speak() override { cout << "汪汪汪"; }
    void eat() override { cout << "吃狗粮"; }
};

// Dog 是具体类，可以创建对象
Dog *d = new Dog();  // 正确
```

## 五、为什么需要多态？

### 5.1 实际应用场景

```cpp
// 场景：游戏引擎需要管理100种角色
class Character {
public:
    virtual void attack() = 0;
    virtual void useSkill() = 0;
    virtual void die() = 0;
};

class Warrior : public Character { /* 实现攻击、技能、死亡 */ };
class Mage : public Character { /* 实现攻击、技能、死亡 */ };
class Archer : public Character { /* 实现攻击、技能、死亡 */ };
// ... 100种角色

// 游戏引擎只需要一个函数处理所有角色
void processBattle(Character* enemies[], int n) {
    for (int i = 0; i < n; i++) {
        enemies[i]->attack();      // 自动调用对应类型的攻击
        enemies[i]->useSkill();    // 自动调用对应类型的技能
        // 不需要判断 enemies[i] 是什么类型！
    }
}
```

### 5.2 图示多态的优势

```
不用多态（100种角色）：
    if (type == DOG) dog->speak();
    else if (type == CAT) cat->speak();
    else if (type == BIRD) bird->speak();
    else if (type == ...  // 写100个else！

用多态（1个函数）：
    animal->speak();  // 自动调用正确的版本！
    // 不管是什么动物，都能正确叫
```

## 六、虚函数的底层原理

### 6.1 你需要记住的

```
没有 virtual 时：
    animal->speak();
    → 编译器直接绑定到 Animal::speak()
    → 永远是父类的版本

有 virtual 时：
    animal->speak();
    → 运行时通过虚表找到实际的函数地址
    → animal 是 Dog 就调用 Dog::speak()
    → animal 是 Cat 就调用 Cat::speak()
```

### 6.2 面试怎么答

> **问题**：为什么要用虚函数？父类的虚函数是空实现，为什么还要写？
>
> **标准答案**：
> ```
> 1. 为了实现多态：不同对象调用同一函数，执行不同行为
>
> 2. 父类定义"规矩"：虚函数在父类里定义接口，告诉所有子类
>    "你们必须实现这个功能"，但父类本身不关心具体怎么实现
>
> 3. 典型应用场景：
>    - 游戏引擎：所有角色继承 Character 基类，都有 attack() 方法
>    - 图形引擎：所有形状继承 Shape 基类，都有 draw() 方法
>    - 排序算法：所有类型实现 compare() 方法
>
> 4. 用法：
>    Animal *p = new Dog();
>    p->speak();  // 调用的是 Dog::speak()，不是 Animal::speak()
> ```

## 七、虚析构函数（专门讲）

### 7.1 问题是这样的

```cpp
class Base {
public:
    virtual void doSomething() {
        cout << "Base doing" << endl;
    }
    // 没有虚析构函数
};

class Derived : public Base {
public:
    ~Derived() {
        cout << "Derived destructor" << endl;  // 这行可能不会执行！
    }
};

int main() {
    Base *p = new Derived();  // 用基类指针指向子类对象
    delete p;  // 只调用 Base 的析构函数！
               // Derived 的析构函数可能不调用！
    return 0;
}
```

### 7.2 为什么会出错？

```
new Derived() 时：
    创建了 Base 部分 + Derived 部分

delete p 时：
    p 是 Base* 类型，只知道要清理 Base 部分
    不知道 Derived 部分存在

结果：
    Derived 部分的资源泄漏了！
```

### 7.3 解决方法：虚析构函数

```cpp
class Base {
public:
    virtual void doSomething() { }
    virtual ~Base() { }  // 加 virtual
};

class Derived : public Base {
public:
    ~Derived() { }  // 自动变成虚析构函数
};

int main() {
    Base *p = new Derived();
    delete p;  // 现在会调用 Derived析构 → Base析构
}
```

### 7.4 图解

```
虚析构函数前：
    delete p
    → 只调用 ~Base()     ← 只释放 Base 部分
                              Derived 部分泄漏！

虚析构函数后：
    delete p
    → 调用 ~Derived()    ← 先调用子类的
    → 调用 ~Base()       ← 再调用父类的
                            完整释放！
```

### 7.5 面试怎么答

> **问题**：什么时候需要虚析构函数？
>
> **标准答案**：
> ```
> 当用基类指针 delete 子类对象时，必须把基类的析构函数设为 virtual。
>
> 原因：
> 普通析构函数是静态绑定，delete 基类指针时只调用基类析构函数。
> 虚析构函数是动态绑定，通过虚表找到实际的析构函数，
> 先调用子类析构，再调用基类析构，完整释放资源。
>
> 记忆口诀：
> "有虚函数要delete子类对象，析构函数必须是virtual"
> ```

## 八、总结：虚函数四问

### Q1: 为什么要用虚函数？
> **A**: 实现多态，一个函数调用，自动执行正确版本

### Q2: 父类虚函数是空的，为什么还要写？
> **A**: 定义"规矩/接口"，告诉子类必须实现这个功能

### Q3: 纯虚函数 (= 0) 和普通虚函数有什么区别？
> **A**: 纯虚函数没有默认实现，子类必须实现；普通虚函数有默认实现，子类可以不写

### Q4: 什么情况下必须用虚析构函数？
> **A**: 用基类指针 delete 子类对象时，必须用虚析构函数
