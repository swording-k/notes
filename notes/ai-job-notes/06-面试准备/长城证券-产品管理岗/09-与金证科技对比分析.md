# 长城证券 vs 金证科技 对比分析

## 一、核心区别：甲方 vs 乙方

```
【乙方】技术供应商         【甲方】金融机构

金证科技 ──────────→ 长城证券、中信证券
恒生电子 ──────────→  银行、基金、保险
宇信科技 ──────────→   ...

（卖技术系统）         （用技术做业务）
```

- **金证科技** = 卖"工具"给金融机构的科技公司
- **长城证券** = 用技术做金融业务的公司

## 三、工作内容差异

### 金证科技（乙方）
```
项目1: 给中信证券做交易系统升级
  → 驻场客户公司
  → 996赶项目
  → 甲方提需求，你来实现

项目2: 给基金公司做TA系统
  → 又是新客户
  → 又要从0理解业务

项目3: 给银行做理财系统
  → 不同业务，又得重新学

优点:
- 接触多家金融机构，业务面广
- 技术深度强（C++/Java底层）
- 薪资高

缺点:
- 长期驻场，节奏紧张
- 没有自家业务积累
- 项目结束可能换地方
```

### 长城证券（甲方）
```
1. 给自家APP做产品优化
  → 一直在同一家公司
  → 业务理解越来越深

2. 给业务部门做系统
  → 跟业务方一起讨论需求
  → 推动功能上线

3. 探索AI在金融的应用
  → 长期投入
  → 慢慢看到效果

优点:
- 业务理解越来越深
- 工作节奏稳定
- 长期积累

缺点:
- 接触面较窄（只看自家业务）
- 薪资天花板较低
- 创新可能较慢
```

## 四、薪资对比

| 工作年限 | 金证科技 | 长城证券 | 互联网大厂 |
|----------|---------|---------|-----------|
| 应届生 | 15-30万 | 8-15万 | 15-25万 |

| 5年 | 40-60万 | 30-50万 | 50-80万 |
| 10年 | 60-100万 | 50-100万 | 100-200万 |

### 关键差异
```
金证科技: 短期高薪，技术深
长城证券: 长期稳定，天花板中等
互联网: 短期高薪，天花板高，35岁危机
```

## 五、适合人群对比

```
金证科技适合:
✅ 想短期赚高薪
✅ 想技术深度成长
✅ 能接受加班
✅ 想接触多家金融机构
✅ 不在意稳定（民营）

长城证券适合:
✅ 想稳定长期发展
✅ 想复合背景（金融+AI）
✅ 看重工作生活平衡
✅ 想进央国企体系
✅ 不在意短期高薪
```

## 六、面试时怎么用"金证面试经验"作为加分项

### 自我介绍可以这样说
> "我对金融科技赛道非常感兴趣。之前面试过金证科技，对**金融IT行业**有了基本了解；现在面试长城证券，是希望**从乙方视角转向甲方视角**，深入理解一家券商的产品研发全流程。"

### 反向提问可以这样问
> "我之前有接触过金证科技这类金融IT公司，他们更偏技术实现；长城证券作为**甲方**，在产品设计上的**自主权**会大很多。请问团队的产品经理在产品决策上有多大空间？"

### 谈薪资时
> "我了解到金证科技校招最高30万。考虑到长城证券是**央企背景**和**深圳生活成本**，我对薪资的预期是**符合央企校招市场水平**。"

## 七、最终总结

```
金证科技 = 民营 + 乙方 + 高薪 + 技术深 + 加班多
长城证券 = 央企 + 甲方 + 稳定 + 复合 + 加班少

两者不是"哪个好"，而是"适合不同的人"
```

**你的情况**：
- 计算机科班 + AI技术 + 想复合背景
- **长城证券更匹配** ✅
- 但如果**金证科技给的产品岗**也不错，可以对比一下
