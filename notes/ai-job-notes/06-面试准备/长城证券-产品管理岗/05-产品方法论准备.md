# 产品方法论准备

> 30分钟掌握产品经理基本功

## 一、PRD怎么写

### PRD是什么

```
PRD = Product Requirements Document = 产品需求文档
作用: 把产品需求讲清楚，让研发、设计、测试都能理解
```

### PRD核心结构

```markdown
# 产品需求文档

## 1. 背景与目标
- 为什么做这个需求
- 业务价值是什么
- 核心指标是什么

## 2. 用户与场景
- 目标用户是谁
- 使用场景是什么
- 解决什么问题

## 3. 功能详情
- 功能列表
- 每个功能的详细说明
- 业务流程图
- 原型图

## 4. 异常情况
- 边界情况
- 错误处理
- 降级方案

## 5. 数据埋点
- 需要采集的数据
- 上报方式
```

### 实际PRD模板（券商场景）

```markdown
# 智能客服升级PRD

## 1. 背景
当前客服人工坐席成本高，响应速度慢，
用户满意度仅70%。

## 2. 目标
- 7×24小时响应
- 解决80%常见问题
- 人工成本降低50%

## 3. 用户
- 25-45岁投资用户
- 移动端为主
- 咨询业务以账户、交易为主

## 4. 功能
4.1 智能问答
- 支持自然语言提问
- 多轮对话
- 知识库覆盖100+常见问题

4.2 人工兜底
- 识别情绪升级
- 复杂问题转人工

## 5. 流程
[流程图]

## 6. 异常
- 网络异常：本地缓存
- 识别失败：转人工

## 7. 埋点
- 提问量
- 解决率
- 满意度
```

## 二、竞品分析怎么做

### 竞品分析的4个步骤

```
1. 确定目标
   - 我要解决什么问题
   - 找什么样的竞品

2. 选择竞品
   - 直接竞品（同业务）
   - 间接竞品（相似功能）
   - 行业标杆（最佳实践）

3. 维度分析
   - 功能对比
   - 体验对比
   - 商业模式对比
   - 技术对比

4. 总结洞察
   - 借鉴什么
   - 避免什么
   - 差异化机会
```

### 券商APP竞品分析框架

```markdown
## 竞品分析：同花顺 vs 东方财富 vs 涨乐财富通

### 维度1：核心功能
| 功能 | 同花顺 | 东方财富 | 涨乐财富通 |
|------|--------|---------|-----------|
| 交易 | ✅ | ✅ | ✅ |
| 行情 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| 资讯 | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 理财 | ⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |
| AI助手 | ⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ |

### 维度2：用户体验
- 同花顺：功能全，UI专业
- 东方财富：内容丰富，社区化
- 涨乐财富通：交易流畅

### 维度3：AI能力

- 东方财富Choice数据
- 涨乐财富通华泰GPT

### 总结
- AI能力是行业趋势
- 智能投顾是差异化机会
- 移动端体验是关键
```

## 四、用户研究方法

### 1. 用户访谈

```
流程:
1. 准备访谈提纲
2. 招募用户（5-10人）
3. 一对一访谈（30-60分钟）
4. 整理洞察

注意:
- 不要引导用户
- 多问"为什么"
- 关注行为而非态度
```

### 2. 问卷调查

```
适用: 大量用户调研
步骤:
1. 设计问卷
2. 投放（200+样本）
3. 数据分析
4. 输出洞察
```

### 3. 数据分析

```
工具: 友盟、神策、GA
方法:
- 漏斗分析（看转化）
- 留存分析（看粘性）
- 行为路径（看流程）
- A/B测试（看效果）
```

### 4. 用户画像

```
要素:
- 基础属性（年龄、性别、职业）
- 投资偏好（风险偏好、资产规模）
- 行为特征（交易频率、APP使用时长）
- 痛点需求
```

## 五、项目管理方法

### 1. 需求优先级

```
RICE模型:
- Reach: 影响用户数
- Impact: 影响程度
- Confidence: 信心程度
- Effort: 投入成本

优先级 = (R × I × C) / E
```

### 2. 版本规划

```
版本节奏:
- 大版本（季度）：重大功能
- 小版本（月度）：优化迭代
- 紧急版本（随时）：bug修复

工具: TAPD、 Jira、 禅道
```

### 3. 跨团队协作

```
技巧:
- 文档先行：PRD、接口文档
- 定期同步：周会、站会
- 拉群沟通：快速对齐
- 拉通对齐：关键决策要确认
```

## 六、面试怎么说

### Q: 你写过PRD吗？

> "写过。在芒果TV产品实习中，我写过XX功能的PRD，结构包括：
> 1. 背景与目标
> 2. 用户与场景
> 3. 功能详情（含流程图和原型）
> 4. 异常处理
> 5. 数据埋点
>
> 写PRD的核心是把需求讲清楚，让研发、设计、测试都能理解。我会特别关注异常情况的处理，避免上线后出问题。"

### Q: 你做过竞品分析吗？

> "做过。我对同花顺、东方财富、涨乐财富通做过分析。从功能、体验、AI能力等维度对比，提炼出对我们有借鉴价值的洞察。比如我发现同花顺的i问财AI能力很强，是行业标杆；东方财富的内容生态做得好；涨乐财富通的交易体验流畅。这些都给了我做产品设计的启发。"

### Q: 你怎么协调跨团队？

> "核心是**文档先行 + 定期同步 + 拉通对齐**。
> 1. 写好PRD，让所有团队有共识
> 2. 每周开站会同步进度
> 3. 关键决策拉群讨论，确认后再推进
>
> 如果遇到冲突，我会拉会议面对面讨论，找到对业务最有利的方案。"
