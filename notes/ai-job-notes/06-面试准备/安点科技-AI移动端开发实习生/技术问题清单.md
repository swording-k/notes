```kotlin
// Q1: Kotlin vs Java 的核心区别？

✅ 参考答案：
"1. 空安全：Kotlin有非空类型系统，编译时避免NPE
2. 扩展函数：无需继承即可给类添加方法
3. 协程：轻量级线程，比Handler更优雅地处理异步
4. 数据类：自动生成equals/hashCode/toString
5. 密封类：更安全的类型分支处理
6. 延迟初始化：by lazy, lateinit var"

---

// Q2: Kotlin 协程是什么？和线程有什么区别？

✅ 参考答案：
"Kotlin协程是轻量级线程，核心概念：

1️⃣ 挂起函数 (suspend)：
   - 可以在执行中途暂停，不会阻塞线程
   - 恢复时从暂停点继续执行

2️⃣ 协程 vs 线程：
   - 线程：创建成本高，切换开销大
   - 协程：可以创建上百万个（因为是用户态调度）
   - 同一线程可以运行多个协程

3️⃣ 常用协程作用域：
   - GlobalScope：全局，慎用
   - viewModelScope：ViewModel生命周期
   - lifecycleScope：Activity/Fragment生命周期

4️⃣ 协程上下文：
   - Dispatchers.Main：主线程（UI更新）
   - Dispatchers.IO：IO密集型（网络/文件）
   - Dispatchers.Default：CPU密集型"

---

// Q3: 什么是扩展函数？

✅ 参考答案：
"扩展函数让你给现有类添加方法而不继承它：

fun String.addExclamation() = this + "!"

// 使用
val str = "hello".addExclamation() // "hello!"

原理：编译成静态方法，String.addExclamation(this)

应用场景：
- 给第三方类添加方法
- 代码风格统一（类似Swift的extension）"

---

// Q4: let/also/run/with 的区别？

✅ 参考答案：
"所有都是作用域函数，区别在于返回值和this指向：

┌──────────┬────────────┬────────────┐
│ 函数     │ this指向   │ 返回值     │
├──────────┼────────────┼────────────┤
│ let      │ it         │ Lambda结果 │
│ also     │ it         │ 对象本身   │
│ run      │ this       │ Lambda结果 │
│ with     │ this       │ Lambda结果 │
│ apply    │ this       │ 对象本身   │
└──────────┴────────────┴────────────┘

使用建议：
- let：空检查 + 链式调用
- also：额外操作（如日志）
- run/run with：需要返回结果
- apply：配置对象"

---

// Q5: Kotlin 的空安全机制？

✅ 参考答案：
"1️⃣ 可空类型：
   val name: String? = null // ? 表示可空

2️⃣ 安全调用操作符 ?.：
   val len = name?.length // name为null时返回null

3️⃣ Elvis操作符 ?:：
   val len = name?.length ?: 0 // 为null时返回0

4️⃣ 非空断言 !!：
   val len = name!!.length // 抛出NPE如果为null

5️⃣ 延迟初始化：
   lateinit var view: TextView // 运行时初始化
   by lazy { ... } // 首次访问时计算"

// Q7: Flutter 的渲染原理？

✅ 参考答案：
"Flutter通过自绘引擎渲染UI，不依赖原生控件：

1️⃣ Widget树：
   - 所有UI都是Widget（可理解为"配置")
   - Widget不可变，状态变化创建新Widget

2️⃣ Element树：
   - Widget的实例化
   - 承上启下：管理Widget和RenderObject

3️⃣ RenderObject树：
   - 真正的渲染对象
   - 计算布局、绘制画面

4️⃣ 光栅化：
   - Skia图形库执行绘制
   - 输出到GPU显存

核心优势：60fps高性能，因为所有UI在C++层绘制"

// Q8: Dart 是单线程模型？如何运行？

✅ 参考答案：
"Dart在单线程中以消息循环机制运行：

┌─────────────────────────────────────────┐
│          微任务队列 (microtask)         │
│    高优先级，如：Future完成回调          │
└──────────────────┬──────────────────────┘
                   ▼
┌─────────────────────────────────────────┐
│          事件队列 (event queue)         │
│    来自UI、计时器、网络等事件           │
└──────────────────┬──────────────────────┘
                   │
              循环往复

1. main()执行完后启动消息循环
2. 先执行所有微任务
3. 再执行一个事件任务
4. 重复上述过程

如何异步：把代码放入微任务队列或事件队列"

---

// Q9: Dart 的级联操作符 .. 是什么？

✅ 参考答案：
"级联操作符返回this，方便链式调用：

// 不使用级联
var paint = Paint();
paint.color = Colors.black;
paint.strokeCap = StrokeCap.round;
paint.strokeWidth = 5.0;

// 使用级联
var paint = Paint()
  ..color = Colors.black
  ..strokeCap = StrokeCap.round
  ..strokeWidth = 5.0;

// 和.的区别：.返回方法返回值，..返回this"
```

// Q16: 什么是RESTful API？

✅ 参考答案：
"REST = Representational State Transfer

核心原则：

1️⃣ 资源导向：
   - 用名词不用动词：/users 而非 /getUsers
   - 资源用URL路径表示

2️⃣ HTTP方法语义化：
   GET    /users      → 获取用户列表
   GET    /users/123  → 获取单个用户
   POST   /users      → 创建用户
   PUT    /users/123  → 更新用户（全量）
   PATCH  /users/123  → 更新用户（部分）
   DELETE /users/123  → 删除用户

3️⃣ 状态码规范：
   200 OK
   201 Created
   400 Bad Request
   404 Not Found
   500 Internal Error

4️⃣ 无状态：
   - 每个请求包含所有必要信息
   - 服务端不存储客户端状态"
```

// Q17: 手写：反转链表

✅ 参考答案：
class ListNode(var `val`: Int) {
    var next: ListNode? = null
}

fun reverseList(head: ListNode?): ListNode? {
    var prev: ListNode? = null
    var curr = head

    while (curr != null) {
        val next = curr.next  // 保存下一个节点
        curr.next = prev      // 反转指针
        prev = curr           // prev前移
        curr = next           // curr前移
    }
    return prev
}

// 时间复杂度：O(n)
// 空间复杂度：O(1)"
```
