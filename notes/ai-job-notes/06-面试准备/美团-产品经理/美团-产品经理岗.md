## 四、项目描述模板（FitGenius）

**STAR法则**：

```
S（Situation）：
我在实习/学习中，发现健身用户面临"不知道练什么、动作不标准"的问题。

T（Task）：
需要做一个能个性化生成训练计划、提供动作指导的产品。

A（Action）：
我独立开发了FitGenius AI健身助手，主要工作：
1. 集成阿里云LLM API，实现自然语言交互
2. 设计多轮对话生成训练计划
3. 实现拍照识别食物、计算机体形态
4. 独立完成iOS开发、App Store上线

R（Result）：
产品已上架App Store，获得XX用户，收到XX反馈。
这个项目让我理解了：
- 怎么用AI能力解决实际问题
- 怎么做用户调研和需求分析
- 怎么快速迭代产品
```

## 五、AI产品经理面试题

### Q: 你怎么理解AI产品经理这个岗位？

```
"AI产品经理的核心是：
1. 发现可以用AI解决的场景
2. 评估AI能力的边界和局限
3. 设计人机交互的方式
4. 保证AI输出质量

和普通产品经理的区别是：
要更懂AI能做什么、不能做什么，
在'AI能力'和'用户需求'之间找到平衡。"
```

### Q: 你觉得AI会怎么改变本地生活？

```
结合美团业务：
1. 外卖：AI推荐更精准，减少选择困难
2. 到店：AI客服解答更及时
3. 闪购：AI预测需求，提前备货
4. 打车：AI调度更高效

核心：AI降低服务成本，提升体验
```

### Q: 你的FitGenius用了什么AI能力？

```
1. LLM对话：生成个性化训练计划
2. 多模态识别：
   - 拍照识别食物热量
   - 视频分析动作标准
3. 上下文理解：记住用户身体状况、训练历史
```
