# 提示工程（Prompt Engineering）

> 用 Prompt 控制 AI 输出，AI产品经理的必备技能

---

## 一、什么是提示工程？

### 一句话定义
```
提示工程 = 如何写好给大模型的指令，让它输出你想要的结果
```

### 类比理解
```
传统开发：写代码 → 程序执行
AI开发：写 Prompt → 模型执行

Prompt 就像是给 AI 写的"代码"，写得好不好直接影响结果
```

---

## 二、提示工程基本原则

### 1. 清晰具体的指令
```
❌ 模糊："帮我写点东西"
✅ 清晰："帮我写一封求职邮件，要求：
   - 简洁，不超过200字
   - 突出Python和算法能力
   - 语气专业但有亲和力"
```

### 2. 给示例（Few-shot）
```
❌ 直接问："把中文翻译成英文"
✅ 给示例：
   "你好=hello
    世界=world
    请翻译：今天天气很好"
```

### 3. 分解复杂任务
```
❌ 一个 Prompt 做复杂任务
✅ Chain of Thought：一步步引导模型思考

示例：
"这是一个数学问题，请分步骤解答：
 Step 1: ...
 Step 2: ...
 最后给出答案"
```

### 4. 指定输出格式
```
要求："用 JSON 格式输出，包含 name、age、skills 三个字段"
要求："用表格形式呈现，每个字段占一列"
```

## 三、常用提示技巧

### 技巧1：角色扮演
```python
prompt = """你是一个有10年经验的AI产品经理，
请用专业但易懂的语言解释什么是RAG。
控制在100字以内。"""
```

### 技巧2：输出约束
```python
prompt = """请列出学习Python的5个步骤，
每步骤不超过20字，用数字编号。"""
```

### 技巧3：思维链（CoT）
```python
prompt = """遇到数学题时，请先展示推理过程，
最后再给出答案。例如：
问题：2+3*4=?
思考：先算乘法 3*4=12
思考：再算加法 2+12=14
答案：14"""
```

### 技巧4：防止幻觉
```python
prompt = """请在回答前检查以下内容：
1. 你确定这个信息是正确的吗？
2. 如果不确定，请说"我不确定"
3. 只输出你确定的内容"""
```

## 四、提示工程 vs 传统规则

| 维度 | 传统规则 | 提示工程 |
|------|---------|---------|
| 定义行为 | 用代码写 if-else | 用自然语言描述 |
| 灵活性 | 固定，扩展困难 | 灵活，易调整 |
| 边界控制 | 精确但刻板 | 自然但需技巧 |
| 维护成本 | 高（代码复杂）| 低（改文字即可）|

## 五、AI产品经理如何用好提示工程

### 场景1：对话产品设计
```
设计 Prompt 引导模型按期望方式回复
控制输出风格、内容长度、专业程度
```

### 场景2：AI效果评估
```
用标准化 Prompt 测试不同模型
对比输出质量、稳定性和响应速度
```

### 场景3：Agent 编排
```
为 Agent 编写系统 Prompt
定义角色、能力边界、行为规范
```

### Q: 什么是提示工程？为什么重要？

```

重要性：
- 同样的模型，好的 Prompt 能提升30%+效果

- 是 AI 产品经理的核心技能之一

我理解好的 Prompt 需要：清晰具体、给示例、分解任务、指定格式
```
