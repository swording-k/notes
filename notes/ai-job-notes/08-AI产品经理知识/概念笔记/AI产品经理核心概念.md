# AI产品经理核心概念

> AI产品经理 = 产品经理能力 + AI技术理解力

---

## 一、AI产品经理 vs 普通产品经理

| 维度   | 普通产品经理          | AI产品经理            |
| ---- | --------------- | ----------------- |
| 核心技能 | 需求分析、PRD撰写、项目管理 | 产品能力 + AI理解力      |
| 设计工具 | 写详细需求文档         | 用 Prompt 控制 AI    |
| 协调对象 | 协调开发团队          | 协调 AI（模型调参、RAG设计） |
| 迭代方式 | 功能迭代            | 效果迭代（准确率、幻觉比例）    |

```
业务理解 → AI边界判断 → 可行性评估 → 需求验证
```

### 2. 功能设计能力

```

清晰的文档结构 + AI效果评估标准
```

| 技术 | 掌握程度 |

| RAG | **重点掌握** |

## 三、AI产品设计模式

### 发散式交互 vs 约束式交互

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **发散式交互** | 自由对话，输入不受限 | AI助手、聊天机器人 |
| **约束式交互** | 按钮/模板/菜单中让用户选择 | 需要稳定输出的场景 |

#### 如何选择？
```
输出稳定性要求高 → 约束式
用户自由度要求高 → 发散式
AI能力强 → 可采用发散式
复杂流程 → 约束式
```

## 四、AI产品生命周期

```
阶段1：需求分析
    ↓ 理解业务痛点、确定AI能力边界、分析可行性

阶段2：产品设计
    ↓ 功能设计、交互设计、输出PRD、制定评估标准

阶段3：开发与训练
    ↓ 协调技术团队、跟踪进度、效果评估

阶段4：上线与迭代
    ↓ 数据分析、用户反馈、优化迭代
```

## 五、面试核心问题

### Q: 如何选择合适的AI模型？
```
分三步：
1. 明确业务场景需求（延迟要求、并发量、成本预算）
2. 测试多款模型的实际表现（不是只看指标）
3. 综合评估后做决策

例如：C端对话场景，优先考虑延迟和成本，复杂任务走专项模型
```

### Q: 如何避免AI幻觉？
```
1. 在工作流中加入反思/检查节点
2. 设置触发阈值，高风险输出人工复核
3. 建立评测体系，持续监控幻觉比例
```

### Q: 模型选型常见错误？
```
错误：只比较技术指标（参数量、benchmark分数）
正确：结合业务场景选择模型
```
