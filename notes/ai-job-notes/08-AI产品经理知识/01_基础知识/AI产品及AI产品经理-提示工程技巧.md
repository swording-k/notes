## 提示工程的核心作用

| 核心价值 | 说明 |
|----------|------|
| **精确引导** | 避免大模型乱答、幻觉、废话 |
| **控制输出** | 让AI按你想要的方式回答 |

> 提示工程不是问问题，而是让AI"按你想的方式回答"

## 结构完整的Prompt示例

```markdown
你是：[角色设定]
一名经验丰富的用户反馈分析师，负责处理电商平台上的用户评论。

任务目标：
- 判断该评论的情感倾向（正面/中性/负面）
- 提取评论中提到的关键词
- 若评论为负面，请写出用户不满意的具体原因
- 输出结果需严格按照指定 JSON 格式

输出格式（JSON）：
{
  "情感倾向": "",
  "关键词": [],
  "总结": "",
  "负面原因": "" // 若为正面或中性，输出 null
}

示例1：
输入评价："这双鞋子真的太舒服了，穿一天都不累，颜色也很好看。"
输出：
{
  "情感倾向": "正面",
  "关键词": ["鞋子", "舒服", "颜色好看"],
  "总结": "用户对鞋子的舒适性和外观表示满意。",
  "负面原因": null
}

示例2：
输入评价："鞋子看起来不错，但穿久了脚底很痛，而且快递也太慢了。"
输出：
{
  "情感倾向": "负面",
  "关键词": ["鞋子", "脚底痛", "快递慢"],
  "总结": "用户对鞋子的舒适度不满意，并对物流速度提出批评。",
  "负面原因": "鞋子穿久脚底痛，快递慢"
}

请一步步思考，先判断整体情绪，再提取关键词，再组织总结。
```

## One Shot / Zero Shot / Few Shot

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| **Zero-shot** | 不给例子，直接说任务 | 简单任务，模型能力强 |
| **One-shot** | 给1个例子 | 需要示范格式/风格 |
| **Few-shot** | 给多个例子 | 复杂任务，需要更多指导 |

### 示例：让大模型改作文

**Zero-shot（不告诉模型任何例子）：**
```
"请给下面这篇作文打分并说明理由。"
{作文内容}
```

**One-shot（先告诉模型一个例子）：**
```
"请给下面这篇作文打分并说明理由。"
参考示例：
作文1：天气很热，我走在回家的路上...
得分：88分，理由：内容具体，结构清晰。

{目标作文内容}
```

**Few-shot（给多个例子）：**
```
作文1 + 得分+理由
作文2 + 得分+理由
作文3 + 得分+理由
→ 作文4：请打分并说明理由
```

> 关键：你给的示例质量高、结构清晰，模型就学得准

## 为什么要用JSON格式输出？

| 原因 | 说明 |
|------|------|
| **机器容易读懂** | 一行代码解析成字典、对象、数据库字段 |
| **结构清晰** | 可直接提取"关键词"字段生成标签，提取"情感倾向"判断正负面 |
| **避免废话** | 如果不指定格式，模型可能输出自由描述，机器不好用 |

### JSON基本结构

```json
{
  "情感倾向": "负面",
  "关键词": ["包装", "客服"],
  "总结": "用户对包装和客服态度不满意。",
  "负面原因": "包装压坏，客服冷淡"
}
```

- 字段（键）+ 值组成，如 `"情感倾向": "负面"`
- 值可以是文字（字符串）、数字、列表、true/false、null等

## 动态Prompt注入业务数据

通过变量动态填充上游数据，适用于跨系统调用：

```markdown
你是：[角色设定]
经验丰富的用户反馈分析师

根据以下业务数据完成用户评价分析：
- 商品名称：{{product_name}}
- 评论内容：{{comment_text}}
- 评论时间：{{comment_time}}
- 商品类目：{{product_category}}

输出格式（JSON）：
{
  "商品": "{{product_name}}",
  "情感倾向": "",
  "关键词": [],
  "总结": "",
  "负面原因": "" // 若为正面或中性，输出 null
}

请一步步思考，先判断整体情绪，再提取关键词，再组织总结。
```

### 输出示例

```json
{
  "商品": "Air Max 跑鞋",
  "情感倾向": "负面",
  "关键词": ["快递", "包装压坏", "客服态度冷淡"],
  "总结": "用户对物流包装和客服服务表达不满。",
  "负面原因": "包装压坏，客服冷淡"
}
```

## 提示工程技巧总结

| 技巧 | 说明 |
|------|------|
| **角色设定** | 明确AI扮演什么角色 |
| **任务目标** | 具体说明要做什么 |
| **输出格式** | 指定JSON等结构化格式 |
| **示例辅助** | One-shot/Few-shot提高准确性 |
| **变量注入** | {{变量名}}动态填充数据 |
| **分步思考** | "请一步步思考"引导推理 |
