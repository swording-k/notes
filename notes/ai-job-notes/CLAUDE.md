## 工作习惯

### 日常学习
1. 在 **Trae** 中写代码（Claude 辅助）
2. 在 Obsidian 日记里记录笔记
3. 晚上把日记整理到对应笔记

### 刷题流程
1. LeetCode 函数模式：理解算法思路
2. Trae ACM 模式：练习输入输出
3. 写完发给我（Claude）：整理到仓库

### Obsidian 笔记规范（重要！）
**所有笔记必须使用 Obsidian 内部链接格式**，这样才能在 Obsidian 中点击跳转。

```markdown
# 正确格式
[[笔记名]]                    # 链接同目录笔记
[[文件夹/笔记名]]              # 链接子目录笔记
[[../上级目录/笔记名]]         # 链接上级目录笔记
![[截图名.png]]               # 嵌入图片

# 示例
今天学了 [[两数之和]]，用到了 [[哈希表]] 的思想。
详见 [[../04-代码练习/刷题清单]]

# 错误格式（禁止使用）
- [文字](链接)              # 这是网页链接，不是内部链接
- /绝对路径.md               # 绝对路径不能用
```

**Claude Code 写笔记时必须**：
- 用 `[[]]` 代替普通链接
- 用相对路径（如 `../`）而非绝对路径
- 嵌入截图用 `![[图片.png]]`
- 在 Trae/VSCode 写的笔记，Obsidian 打开后链接依然可点击

## 代码存放规范

### 目录命名
- 每个题目用 **LeetCode 题号** 命名，如 `0198-打家劫舍`、`0070-爬楼梯`
- 格式：`XXXX-题目名`，不足4位前面补0

### 文件结构
每个题目目录下包含：
- `XXXX-题目名.md` - 原题 + 题解（**必须包含原题链接**）
- `XXXX-题目名.py` - 代码模板

### 文档要求（必须遵守）
1. **原题链接**：每个题目文档开头必须包含 `> 原题链接：https://leetcode.cn/problems/xxx/` 格式
2. **题目描述**：完整原始描述
3. **输入输出格式**：题目要求的输入输出
4. **示例**：示例输入输出
5. **思路解析**：状态定义、转移方程、初始化
6. **代码实现**：带注释的完整代码
7. **面试问答**：常见面试问题（如有）

### 刷题流程
1. LeetCode 函数模式：理解算法思路
2. Trae ACM 模式：练习输入输出
3. 写完发给我（Claude）：整理到仓库

## 给 Claude 的建议

- 用户是初学者，解释要详细，代码要拆解
- 用户容易焦虑，需要鼓励 + 具体行动建议
- 优先回答代码问题，再整理笔记
- 发现问题及时指出（比如语法错误）
- **重要**：用户提出的新要求、新规范、聊天中达成的共识，**必须实时更新到 CLAUDE.md 和 README.md**，不能停留在旧版本
- **讲解任何知识时的要求**：
  1. 讲清楚**是什么**、**为什么**、**怎么用**
  2. 知识点要能举一反三，不只是这道题
  3. 细节不跳过，每个步骤都要解释
  4. 讲完要确认用户理解，不理解就再讲
  5. **重点细节必须展开讲**：
     - 虚拟头节点（dummy）为什么需要？解决什么问题？
     - 快慢指针的步长、间隔、为什么这样设置
     - 每一步操作改了什么、指向哪
     - 边界情况怎么处理
     - 用具体数字从头到尾走一遍流程
  6. **知识沉淀**：讨论出的新知识点必须同步到对应笔记（概念笔记或刷题笔记）
  7. **题解 md 文件要求**：
     - 包含原题链接
     - 包含题目描述
     - 思路解析要讲透每个细节，不跳过任何步骤
     - 代码要有注释解释关键行
     - 面试问答（如果有常见面试问题）
