# LeetCode 455. 分发饼干

> 原题链接：https://leetcode.cn/problems/assign-cookies/

## 题目描述

假设你是一位很棒的家长，想要给你的孩子们一些小饼干。但是，每个孩子最多只能给一块饼干。

每个孩子 i 有一个胃口值 `g[i]`，这是让孩子们感到满足的最小饼干尺寸；每块饼干 j 有一个尺寸 `s[j]`。

如果 `s[j] >= g[i]`，我们可以将这个饼干 j 分配给孩子 i ，这个孩子会得到满足。你的目标是尽可能满足越多数量的孩子，并输出这个最大数值。

## 示例

```
示例 1：
输入：g = [1,2,3], s = [1,1]
输出：1

示例 2：
输入：g = [1,2], s = [1,2,3]
输出：2
```

---

## 解题思路

### 方法：贪心算法

**核心思想**：用尽量小的饼干满足尽量多的孩子

**策略**：
- 将孩子胃口和饼干尺寸都排序
- 用小饼干优先满足小胃口，不行则用更大的

---

## 图解

**示例：g = [1,2,3], s = [1,1]**

```
排序后：g = [1,2,3], s = [1,1]

遍历饼干：
┌──────┬──────┬─────────────┬─────────────┐
│ 饼干  │ 尺寸  │ 孩子胃口(g) │ 是否满足    │
├──────┼──────┼─────────────┼─────────────┤
│  1   │  1   │    1        │ ✓ 满足，count=1 │
│  2   │  1   │    2        │ ✗ 不够，换下一个饼干 │
└──────┴──────┴─────────────┴─────────────┘

结果：1
```

**示例：g = [1,2], s = [1,2,3]**

```
排序后：g = [1,2], s = [1,2,3]

遍历饼干：
┌──────┬──────┬─────────────┬─────────────┐
│ 饼干  │ 尺寸  │ 下一个孩子胃口 │ 是否满足    │
├──────┼──────┼─────────────┼─────────────┤
│  1   │  1   │    1        │ ✓ count=1   │
│  2   │  2   │    2        │ ✓ count=2   │
│  3   │  3   │   已满足完   │    -        │
└──────┴──────┴─────────────┴─────────────┘

结果：2
```

## 代码实现

```python
class Solution:
    def findContentChildren(self, g: List[int], s: List[int]) -> int:
        g.sort()  # 孩子胃口排序
        s.sort()  # 饼干尺寸排序

        child = 0  # 已满足的孩子数（也是下一个孩子的索引）

        for size in s:  # 遍历每个饼干
            if child < len(g) and size >= g[child]:
                child += 1  # 这个饼干满足了这个孩子

        return child
```

---

## 代码关键点解释

| 步骤 | 代码 | 解释 |
|------|------|------|
| 1 | `g.sort(); s.sort()` | 先排序，才能用小饼干优先满足 |
| 2 | `for size in s` | 遍历饼干，从小到大 |
| 3 | `size >= g[child]` | 饼干够满足当前孩子 |
| 4 | `child += 1` | 满足了一个孩子，索引后移 |

## 面试问答

**Q：为什么排序后遍历就能保证最优？**

A：用反证法。如果小饼干不满足小胃口，而去满足大胃口，那么小饼干可能浪费（不够大胃口用），大饼干却可以满足大胃口。排序保证了我们总是"尽力"满足当前最需要的孩子。

**Q：可以用大饼干先满足大胃口吗？**

A：可以，思路类似，逆向遍历即可。贪心只要保证"不浪费大饼干"就行。
