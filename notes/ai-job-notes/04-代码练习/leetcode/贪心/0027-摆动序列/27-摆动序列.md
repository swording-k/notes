# LeetCode 376. 摆动序列

> 原题链接：https://leetcode.cn/problems/wiggle-subsequence/

## 题目描述

如果连续序列的差值严格正负交替，则称其为摆动序列。一个序列的子序列可以通过删除一些元素而不改变其余元素的顺序获得。

给你一个整数数组 `nums`，返回 `nums` 中作为摆动序列的最长子序列的长度。

## 示例

```
示例 1：
输入：nums = [1,7,4,9,2,5]
输出：6
解释：整个数组是一个摆动序列，差值 [+6,-3,+5,-7,+3] 交替

示例 2：
输入：nums = [1,4,7,2,5]
输出：5
解释：可以选出 [1,4,7,2,5] 是一个摆动序列

示例 3：
输入：nums = [1,2,3,4,5,6,7,8,9]
输出：2
```

---

## 解题思路

### 方法：贪心算法

**核心思想**：找所有峰和谷，峰谷之间的元素可以删除

**贪心策略**：
- 上坡不拐弯 → 继续延伸
- 出现拐点 → 记录峰或谷
- 中间的元素不需要保留

## 图解

**示例：nums = [1,7,4,9,2,5]**

```
          峰9
         ╱  ╲
    谷4 ╱    ╲  谷2
      ╱      ╲╱
  7 ╱        ╲ 5
   ╱          ╲
 1            ╲
              ↘

摆动：+6, -3, +5, -7, +3  ← 严格正负交替

峰：7→4 下降，9是峰
谷：4→9 上升，4是谷

结果：6（所有峰谷都保留）
```

**示例：nums = [1,4,7,2,5]**

```
       峰7
      ╱ ╲
  4  ╱   ╲ 5
   ╱     ╲╱
 1       2

差值：+3,+3,-5,+3  ← 有连续上升，不是摆动
需要删除 4→7 中间的某个：删除4或7

结果：[1,4,7,2,5] 或 [1,4,2,5]
```

## 代码实现

### 方法一：双指针（直观）

```python
class Solution:
    def wiggleMaxLength(self, nums: List[int]) -> int:
        if len(nums) < 2:
            return len(nums)

        # 差值数组：[1-0, 2-1, 3-2, ...]
        diff = [nums[i] - nums[i-1] for i in range(1, len(nums))]

        result = 1  # 最少是1
        for i in range(len(diff)):
            # 找到第一个非零差值作为起始
            if diff[i] != 0:
                result += 1
                prev = diff[i]
                break

        # 遍历后续差值，找正负交替
        for i in range(i+1, len(diff)):
            if diff[i] * prev < 0:  # 符号相反
                result += 1
                prev = diff[i]

        return result
```

### 方法二：动态规划（空间 O(1)）

```python
class Solution:
    def wiggleMaxLength(self, nums: List[int]) -> int:
        if len(nums) < 2:
            return len(nums)

        up = down = 1  # 以上升/下降结尾的最长摆动长度

        for i in range(1, len(nums)):
            if nums[i] > nums[i-1]:
                up = down + 1    # 上升段长度 = 之前下降段长度 + 1
            elif nums[i] < nums[i-1]:
                down = up + 1    # 下降段长度 = 之前上升段长度 + 1

        return max(up, down)
```

## 代码关键点解释

| 步骤 | 代码 | 解释 |
|------|------|------|
| 1 | `up = down = 1` | 初始化为1（单个元素） |
| 2 | `nums[i] > nums[i-1]` | 上升段 |
| 3 | `up = down + 1` | 上升段长度 = 之前下降段长度 + 1 |
| 4 | `max(up, down)` | 返回较长的摆动序列 |

## 面试问答

**Q：为什么可以用贪心？**

A：因为我们需要的是"正负交替"，只要保留峰和谷就足够了。中间的元素无论是高是低，都不影响摆动性质，可以删掉。

**Q：up 和 down 分别代表什么？**

A：`up` 是以**上升**结尾的最长摆动长度，`down` 是以**下降**结尾的最长摆动长度。每次遇到拐点就更新对应值。
