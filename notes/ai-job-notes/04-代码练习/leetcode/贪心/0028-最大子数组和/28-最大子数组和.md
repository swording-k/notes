# LeetCode 53. 最大子数组和

> 原题链接：https://leetcode.cn/problems/maximum-subarray/

## 题目描述

给你一个整数数组 `nums`，找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

## 示例

```
示例 1：
输入：nums = [-2,1,-3,4,-1,2,1,-5,4]
输出：6
解释：连续子数组 [4,-1,2,1] 的和最大，为 6

示例 2：
输入：nums = [1]
输出：1

示例 3：
输入：nums = [5,4,-1,7,8]
输出：23
```

---

## 解题思路

### 方法：贪心算法

**核心思想**：当连续和为负时，放弃前面的，从当前位置重新开始

**贪心策略**：
- 如果前面的和是负数，对后续没有任何帮助
- 负数 + 任何数 < 那个数本身
- 所以应该从下一个位置重新累积

---

## 图解

**示例：nums = [-2,1,-3,4,-1,2,1,-5,4]**

```
nums:  -2   1  -3   4  -1   2   1  -5   4
         ↘   ↘  ↘   ↘  ↘   ↘   ↘   ↘
cur:    -2   1  -2   4   3   5   6   1   5
max:    -2   1   1   4   4   5   6   6   6

重新开始的时刻：
- cur=-2+1=-1 (负) → 重新开始 → cur=1
- cur=1-3=-2 (负) → 重新开始 → cur=4
- ...

结果：max=6，对应子数组 [4,-1,2,1]
```

**为什么负数和要放弃？**

```
假设：[-2,1] 的和 = -1

如果要加第三个元素 -3：
-1 + (-3) = -4
而直接用 -3 更大

所以前面的负数和确实没用，应该放弃
```

## 代码实现

```python
class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        max_sum = nums[0]    # 全局最大和
        cur_sum = nums[0]   # 当前连续和

        for i in range(1, len(nums)):
            # 贪心：重新开始 vs 继续累加
            # 如果前面是负数，重新开始；否则累加
            cur_sum = max(nums[i], cur_sum + nums[i])
            max_sum = max(max_sum, cur_sum)

        return max_sum
```

---

## 代码关键点解释

| 步骤 | 代码 | 解释 |
|------|------|------|
| 1 | `cur_sum = max(nums[i], cur_sum + nums[i])` | 重新开始 vs 继续 |
| 2 | `max_sum = max(max_sum, cur_sum)` | 更新全局最大 |
| 3 | 初始化为 `nums[0]` | 空数组要处理 |

## 动态规划对比

```python
# 动态规划版本
def maxSubArray(nums):
    dp = [0] * len(nums)
    dp[0] = nums[0]
    for i in range(1, len(nums)):
        dp[i] = max(nums[i], dp[i-1] + nums[i])
    return max(dp)
# 贪心版本只是不需要 dp 数组，直接用变量
```

## 面试问答

**Q：为什么贪心有效？**

A：对于任何以负数开头的子数组，它的和必然小于从后面正数开始的子数组和。所以负数和可以直接放弃。

**Q：和动态规划的区别？**

A：本质一样，都是 `dp[i] = max(nums[i], dp[i-1] + nums[i])`。贪心只是把 dp 数组优化成了一个变量。
