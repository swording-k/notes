# 0279. 完全平方数 / Perfect Squares

> 原题链接：https://leetcode.cn/problems/perfect-squares/

## 题目描述

给你一个整数 `n`，返回和为 `n` 的完全平方数的最少数量。

完全平方数 是一个整数，其值等于另一个整数的平方；换句话说，其值等于一个整数自乘的积。例如，1、4、9 和 16 都是完全平方数，而 3 和 11 不是。

## 示例

```
输入: 12
输出: 3

说明: 12 = 4 + 4 + 4
```

## 数据范围

```
1 <= n <= 10^4
```

---

## 思路解析

### 问题转化

将整数 `n` 看成长度为 `n` 的木板，完全平方数 `x²` 看成长度为 `x²` 的木条。

问题转化为：**将长度为 `n` 的木板分割成最少的部分，要求每个部分的长度恰好是完全平方数。**

### 状态定义

`f(i)` = 长度为 `i` 的木板能分割出的最少完全平方数个数

**边界**：`f(0) = 0`，长度为0的木板不需要切割

### 状态转移

考虑最后一个部分的可能长度：`j²`（j = 1, 2, ..., √i）

```
f(i) = min(f(i - j²) + 1)  for all j where j² <= i
```

- 枚举所有可能的平方数 `j²`
- 减掉这个平方数后，问题变成 `f(i - j²)`
- 加上当前这个平方数

### 计算顺序

因为依赖的状态 `i - j² < i`，所以从前往后计算。

---

## 代码实现

```python
class Solution:
    def numSquares(self, n: int) -> int:
        """
        动态规划解法：
        dp[i] 表示和为 i 的最少完全平方数个数

        转移方程：dp[i] = min(dp[i - j²] + 1) for all j where j² <= i

        时间复杂度：O(n√n)
        空间复杂度：O(n)
        """
        # dp[i] 初始化为正无穷，dp[0] = 0
        dp = [float('inf')] * (n + 1)
        dp[0] = 0

        # 遍历每个数字 i
        for i in range(1, n + 1):
            j = 1
            # 枚举所有可能的完全平方数 j²
            while j * j <= i:
                dp[i] = min(dp[i], dp[i - j * j] + 1)
                j += 1

        return dp[n]


# ================ ACM模式 ================
if __name__ == "__main__":
    n = int(input())
    sol = Solution()
    print(sol.numSquares(n))
```

---

## 空间优化思路

由于 `dp[i]` 只依赖 `dp[i - j²]`，无法用常量优化空间。

但可以预处理所有完全平方数，减少内层循环判断。

## 面试问答

**Q: 这道题的思路是什么？**

A: 动态规划。我们定义 `dp[i]` 表示组成数字 `i` 所需要的最少完全平方数个数。
然后考虑最后一个数一定是某个平方数，所以枚举所有小于等于 `i` 的平方数 `j²`，
把它减掉之后，问题就变成了 `i - j²` 的最优解，也就是 `dp[i - j²]`，
再加上当前这个平方数即可。
初始化 `dp[0] = 0`，因为组成0不需要任何数。
最后答案就是 `dp[n]`。
时间复杂度是 **O(n√n)**，因为外层循环枚举 n 次，内层循环最多枚举 √n 次。
