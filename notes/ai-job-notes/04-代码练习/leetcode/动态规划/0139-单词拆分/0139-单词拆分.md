# 0139. 单词拆分 / Word Break

> 原题链接：https://leetcode.cn/problems/word-break/

## 题目描述

给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。如果可以利用字典中出现的一个或多个单词拼接出 `s` 则返回 `true`。

注意：
- 不要求字典中出现的单词全部都使用
- 字典中的单词可以重复使用

## 示例

```
输入: s = "leetcode", wordDict = ["leet", "code"]
输出: true

说明: 返回 true 因为 "leetcode" 可以由 "leet" 和 "code" 拼接成。
```

---

## 思路解析

### 状态定义

`f(i)` = 字符串 `s` 的前 `i` 个字符（`s[0:i]`）能否由字典中的单词组成

`f(i) ∈ {True, False}`

### 边界条件

`f(0) = True`  # 空字符串可以被成功切分

### 状态转移

枚举切分点 `j`（0 <= j < i），把字符串分成：
- 前缀 `s[0:j]`
- 后缀 `s[j:i]`

如果 `f(j) == True` 且 `s[j:i]` 在字典中，则 `f(i) = True`

```
f(i) = OR(f(j) AND s[j:i] in wordDict)  for all j in [0, i)
```

### 计算顺序

从前往后计算，因为依赖的状态 `f(j)` 始终小于 `i`

### 时间复杂度

```
O(n²)  # n为字符串长度，每个位置枚举所有切分点
```

## 代码实现

```python
class Solution:
    def wordBreak(self, s: str, wordDict: list[str]) -> bool:
        """
        动态规划解法

        dp[i] 表示 s[0:i] 是否可以被切分成字典中的单词

        时间复杂度: O(n²)
        空间复杂度: O(n)
        """
        n = len(s)
        dp = [False] * (n + 1)
        dp[0] = True  # 空串可以被成功切分

        # 遍历字符串每个位置
        for i in range(1, n + 1):
            # 枚举所有可能的切分点 j
            for j in range(i):
                # 如果 s[0:j] 可切分 且 s[j:i] 在字典中
                if dp[j] and s[j:i] in wordDict:
                    dp[i] = True
                    break  # 找到一种切分方式即可退出

        return dp[n]


# ================ ACM模式 ================
if __name__ == "__main__":
    # 第一行：字符串 s
    s = input().strip()
    # 第二行：字典单词列表（空格分隔）
    wordDict = set(input().split())

    sol = Solution()
    result = sol.wordBreak(s, wordDict)
    print("true" if result else "false")
```

## 面试问答

**Q: 这道题的思路是什么？**

A: 动态规划。定义 `dp[i]` 表示字符串前 `i` 个字符能不能被字典里的单词拼出来。
我们去枚举一个切分点 `j`，把字符串分成两部分：前缀 `s[0:j]` 和 后缀 `s[j:i]`。
如果前面这一段是可以拼出来的，并且后面这一段刚好在字典里，那当前这个位置 `i` 就是可以拼出来的。
所以本质就是：枚举最后一段，看看能不能合法切开。
初始化 `dp[0] = True`，表示空字符串是可以被拼出来的。
从小往大进行枚举计算 dp 数组，最后答案就是 `dp[n]` 的结果。
时间复杂度是 **O(n²)**。
