# LeetCode 15. 三数之和

> 原题链接：https://leetcode.cn/problems/3sum/

## 题目描述

给你一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足：
- `i != j`、`i != k`、`j != k`
- `nums[i] + nums[j] + nums[k] == 0`

返回所有满足条件的三元组。注意：答案中不可包含重复的三元组。

## 示例

```
示例 1：
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]

示例 2：
输入：nums = [0,1,2]
输出：[]

示例 3：
输入：nums = []
输出：[]
```

---

## 解题思路

### 方法：排序 + 双指针

**核心思想**：
1. 先排序数组
2. 固定一个数 `k`，用双指针找另外两个数
3. 通过排序后的相邻判断跳过重复元素

**为什么排序有用**：
- 排序后相等的元素会相邻
- 可以通过 `nums[k] == nums[k-1]` 跳过重复组合
- 双指针 O(n²) 比暴力 O(n³) 快

---

## 图解过程

**示例：nums = [-1, 0, 1, 2, -1, -4]**

```
排序后：[-4, -1, -1, 0, 1, 2]

Step 1: k = 0, nums[0] = -4
        i = k+1 = 1, j = len-1 = 5
        [-4, -1, 2] = -3 ≠ 0
        因为 -4 < 0，继续

Step 2: k = 1, nums[1] = -1
        i = 2, j = 5
        [-1, -1, 2] = 0 ✓ 找到！
        添加 [-1,-1,2]

Step 3: k = 2, nums[2] = -1（与 nums[1] 相同，跳过）
        去重：if k > 0 and nums[k] == nums[k-1]: continue

Step 4: k = 3, nums[3] = 0
        i = 4, j = 5
        [0, 1, 2] = 3 ≠ 0
        因为 3 > 0，j-- 再试

Step 5: k = 4, nums[4] = 1
        1 > 0，后面的和都 > 0，break
```

## 代码实现

```python
class Solution:
    def threeSum(self, nums: List[int]) -> List[List[int]]:
        nums.sort()           # 1. 先排序
        result = []

        for k in range(len(nums) - 2):  # k 是固定的第三个数字
            if nums[k] > 0:             # 最小的都大于0，不可能有0
                break
            if k > 0 and nums[k] == nums[k-1]:  # 去重，跳过相同元素
                continue

            i, j = k + 1, len(nums) - 1  # 双指针

            while i < j:
                s = nums[k] + nums[i] + nums[j]
                if s < 0:
                    i += 1              # 和太小，增大 i
                elif s > 0:
                    j -= 1              # 和太大，减小 j
                else:
                    result.append([nums[k], nums[i], nums[j]])
                    i += 1
                    j -= 1
                    while i < j and nums[i] == nums[i-1]:
                        i += 1          # 跳过重复的 i
                    while i < j and nums[j] == nums[j+1]:
                        j -= 1          # 跳过重复的 j

        return result
```

## 代码关键点解释

| 步骤 | 代码 | 解释 |
|------|------|------|
| 1 | `nums.sort()` | 排序是前提 |
| 2 | `for k in range(len(nums)-2)` | k 不能太大，至少留两个位置 |
| 3 | `if nums[k] > 0: break` | 剪枝：小数都正，大数之和肯定 > 0 |
| 4 | `if k > 0 and nums[k] == nums[k-1]: continue` | 去重核心：跳过相同元素 |
| 5 | `i, j = k + 1, len(nums) - 1` | 双指针从 k 之后开始 |
| 6 | `while i < j` | 双指针循环 |
| 7 | `if s < 0: i += 1 / elif s > 0: j -= 1` | 调整指针方向 |

## 面试问答

**Q：为什么排序后可以跳过重复元素？**

A：排序后相等的元素会相邻。如果 `nums[k] == nums[k-1]`，说明这个数已经被当作起点枚举过了，再以它为起点会产生重复的三元组。

**Q：为什么 `nums[k] > 0` 就要 break？**

A：因为数组已排序，`nums[k]` 是当前及之后最小的数。如果它都大于 0，那后面任意三个数之和都大于 0，不可能是 0。

**Q：双指针为什么能找到所有组合？**

A：当 `k` 固定时，`i + j` 的范围是确定的。指针从两端向中间移动，每次根据 `sum` 的大小决定移动哪个指针，能遍历所有可能的 `(i, j)` 组合。
