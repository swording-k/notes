# LeetCode 33. 搜索旋转排序数组

## 题目链接
https://leetcode.cn/problems/search-in-rotated-sorted-array/

## 题目描述
给你一个旋转后的升序数组（原来有序，某个位置旋转了）和一个目标值 target。在数组中查找 target，返回其索引；如果不存在返回 -1。要求时间复杂度 O(log n)。

## 示例
```
示例 1：
输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4
解释：0 在数组中，下标是 4

示例 2：
输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1

示例 3：
输入：nums = [1], target = 0
输出：-1
```

## 提示
- 1 ≤ nums.length ≤ 5000
- nums 按升序排列，值不重复
- 题目保证 nums 在某个位置旋转

## 解题思路

### 变种二分查找 O(log n) ✅
```python
class Solution:
    def search(self, nums: list[int], target: int) -> int:
        if not nums:
            return -1

        left, right = 0, len(nums) - 1

        while left <= right:
            mid = (left + right) // 2

            if nums[mid] == target:
                return mid

            # 判断哪半段是有序的
            if nums[mid] >= nums[left]:
                # 左半段有序
                if nums[left] <= target < nums[mid]:
                    right = mid - 1
                else:
                    left = mid + 1
            else:
                # 右半段有序
                if nums[mid] < target <= nums[right]:
                    left = mid + 1
                else:
                    right = mid - 1

        return -1
```

### 核心思想
- 旋转数组分成两段有序区间
- 关键：判断 mid 在哪一段
- `nums[mid] >= nums[left]` → 左半段有序
- 否则右半段有序

### 判断逻辑图
```
原数组：[0,1,2,4,5,6,7] 旋转点在 4
旋转后：[4,5,6,7,0,1,2]
        ↑左半段有序  ↑右半段有序

mid 在左半段时：target 只能在 nums[left] ~ nums[mid] 之间
mid 在右半段时：target 只能在 nums[mid] ~ nums[right] 之间
```
