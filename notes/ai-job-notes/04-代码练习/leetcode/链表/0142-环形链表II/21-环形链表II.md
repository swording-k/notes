# LeetCode 142. 环形链表 II

> 原题链接：https://leetcode.cn/problems/linked-list-cycle-ii/

## 题目描述

给定一个链表的头节点 `head` ，判断链表是否有环。如果有环，返回环的入口节点；如果没有环，返回 `null`。

## 示例

```
示例 1：
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的节点（即节点2）

示例 2：
输入：head = [1,2], pos = 0
输出：返回索引为 0 的节点（即节点1）
```

---

## 解题思路

### 方法：Floyd 快慢指针 + 数学证明

**核心思想**：快慢指针第一次相遇后，从链表头和相遇点同时同速出发，必然在环入口相遇。

---

## 图解与数学推导

### 第一步：设定变量

```
链表结构：
┌─────────────────────────────────────────────────┐
│                                                 │
head ──a──→ [入口] ←────────c──────── [相遇点]     │
              ↑                    │             │
              │                    │             │
              └────────b───────────┘             │
                                                 │
└─────────────────────────────────────────────────┘

| 符号 | 含义 |
|------|------|
| `a` | 从头到环入口的距离（环前部分） |
| `b` | 从环入口到相遇点的距离 |
| `c` | 从相遇点绕回环入口的距离（`c = 环长 - b`） |
```

### 第二步：第一次相遇

- **慢指针**速度 1，走的距离 = `a + b`
- **快指针**速度 2，走的距离 = `a + b + c + b = a + 2b + c`
  - （快走了完整一圈环：`c`，加上 `a` 到入口和 `b` 到相遇点）

因为快是慢的 2 倍速度：
```
2(a + b) = a + 2b + c
```

化简：
```
2a + 2b = a + 2b + c
2a = a + c
a = c
```

> [!tip] 关键结论
> **`a = c`** — 从头到环入口的距离 = 从相遇点绕环回到入口的距离

### 第三步：第二次相遇找入口

第一次相遇后：
- `slow`（慢指针）在**相遇点**
- `fast`（快指针）重置到**链表头**

然后两人同时走 `a` 步：

| 指针 | 起点 | 走 `a` 步后到达 |
|------|------|----------------|
| `fast` | 链表头 | 环入口 |
| `slow` | 相遇点 | 绕环走 `c` 步（因为 `a = c`）→ 环入口 |

**两人在环入口相遇！**

---

## 具体数字例子

```
链表：1 → 2 → 3 → 4 → 5 → 6
              ↑_______________|

设：
- a = 2（1→2 的距离）
- b = 3（2→3→4→5，到相遇点）
- c = 2（5→6→2，绕回入口）

第一次相遇：
- slow 走了：a + b = 2 + 3 = 5（节点5）
- fast 走了：a + b + c + b = 2 + 3 + 2 + 3 = 10（2圈）
- 验证：10 = 2 × 5 ✓
- a = c = 2 ✓

第二次：
- fast 重置到节点1
- slow 从节点5出发

同时走 a=2 步：
- fast：1 → 2 → 3（环入口）
- slow：5 → 6 → 3（绕环走 c=2 步，也到环入口）

相遇点 = 节点3 = 环入口 ✓
```

---

## 代码实现

```python
class Solution:
    def detectCycle(self, head: ListNode) -> ListNode:
        # 边界检查
        if not head:
            return None

        # ========== 第一步：快慢指针找相遇点 ==========
        fast = head
        slow = head

        while fast and fast.next:
            fast = fast.next.next  # 快走2步
            slow = slow.next       # 慢走1步
            if fast == slow:       # 相遇
                break

        # ========== 第二步：无环检测 ==========
        # 条件：fast 到末尾了（没有环）
        if not fast or not fast.next:
            return None

        # ========== 第三步：找环入口 ==========
        # 将 slow 重置到头，fast 保持在相遇点
        # 两人同速前进，必然在环入口相遇
        slow = head
        while slow != fast:
            slow = slow.next
            fast = fast.next

        return slow  # 环入口
```

---

## 代码关键点解释

| 步骤 | 代码 | 解释 |
|------|------|------|
| 1 | `fast = fast.next.next` | 快走2步 |
| 2 | `slow = slow.next` | 慢走1步 |
| 3 | `if fast == slow: break` | 第一次相遇 |
| 4 | `if not fast or not fast.next` | 无环检测 |
| 5 | `slow = head` | slow 重置到头 |
| 6 | `while slow != fast` | 同速前进找入口 |

## 复杂度分析

| 复杂度 | 值 | 说明 |
|--------|---|------|
| **时间** | O(n) | 第一步最多走 n 步，第二步最多走 n 步 |
| **空间** | O(1) | 只用两个指针 |

## 面试问答

**Q：为什么第一次相遇后 slow 和 fast 同速就能找到入口？**

A：因为数学推导证明了 `a = c`，即从头到入口的距离等于从相遇点绕环回去的距离。两人同速走 `a` 步，必然同时到达入口。

**Q：为什么快指针速度是 2 不是 3 或其他？**

A：快指针速度只要大于慢指针，第一次相遇时就有 `2(a+b) = a + b + k×环长`，可以推导出 `a = 环长 - b`。但 2 是最小最常用的选择。
