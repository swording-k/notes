# LeetCode 22. 组合

> 原题链接：https://leetcode.cn/problems/combinations/

## 题目描述

给定两个整数 `n` 和 `k`，返回范围 `[1, n]` 中所有可能的 `k` 个数的组合。

## 示例

```
示例 1：
输入：n = 4, k = 2
输出：[[1,2], [1,3], [1,4], [2,3], [2,4], [3,4]]

示例 2：
输入：n = 1, k = 1
输出：[[1]]
```

---

## 解题思路

### 方法：回溯算法

**核心思想**：从 1~n 中选择 k 个数，组合不考虑顺序

**关键点**：
- 使用 `start` 参数控制选择范围（不回头选）
- 剪枝优化：剩余元素不够时提前停止

---

## 图解回溯过程

**示例：n=4, k=2**

```
                      []
        ┌─────────────┼─────────────┐
        │             │             │
       [1]           [2]           [3]         ← 第一层：选哪个起始数字
    ┌───┴───┐     ┌───┴───┐     ┌───┴───┐
    │       │     │       │     │       │
  [1,2]   [1,3] [2,3]   [2,4] [3,4]  (无效)
    │       │     │       │     │
  收集    收集   收集   收集  收集

结果：[[1,2], [1,3], [2,3], [2,4], [3,4]]
```

---

## 代码实现

```python
class Solution:
    def combine(self, n: int, k: int) -> List[List[int]]:
        result = []

        def backtrack(start, path):
            # 终止条件：选够 k 个数
            if len(path) == k:
                result.append(path.copy())
                return

            # 遍历选择
            for i in range(start, n + 1):
                # ===== 剪枝优化 =====
                # 如果剩余元素不够，直接停止
                # 剩余元素 = n - i + 1
                # 已有元素 = len(path)
                if len(path) + (n - i + 1) < k:
                    break
                # ====================

                path.append(i)       # 做选择
                backtrack(i + 1, path)  # 递归（i+1，不回头）
                path.pop()           # 撤销选择

        backtrack(1, [])
        return result
```

---

## 代码关键点解释

| 步骤 | 代码 | 解释 |
|------|------|------|
| 1 | `for i in range(start, n + 1)` | 从 start 开始选，不回头 |
| 2 | `if len(path) + (n - i + 1) < k: break` | 剪枝：剩余不够就停 |
| 3 | `backtrack(i + 1, path)` | 递归时传 i+1，不是 start |
| 4 | `path.pop()` | 撤销选择，回溯 |

| 复杂度 | 值 | 说明 |
|--------|---|------|
| **时间** | O(C(n,k) × k) | 组合数 × 复制路径的复杂度 |
| **空间** | O(k) | 递归栈深度 |

## 面试问答

**Q：组合和排列的区别？**

A：组合不考虑顺序，`[1,2]` 和 `[2,1]` 是同一个组合；排列考虑顺序，是不同的。组合用 `start` 参数控制不回头，排列用 `used` 数组标记已使用。

**Q：为什么要剪枝？**

A：剪枝可以提前跳过不可能产生有效解的分支，减少无效搜索。例如 n=100, k=2，当 start=99 时只剩 [99,100] 一个有效组合，后面的可以跳过。
