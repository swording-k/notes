# 哈希表

> 时间复杂度 O(1) 的查找结构，刷题高频神器

---

## 一、什么是哈希表？

### 生活类比
```
图书馆找书：
- 不用书架逐个找 → 用索引目录 → 瞬间定位
- 哈希表就是"索引目录"，用空间换时间
```

### Python 实现
```python
# 字典 = 哈希表
hash_map = {}           # 创建空哈希表
hash_map[key] = value    # 插入/修改
value = hash_map[key]   # O(1) 查找
value = hash_map.get(key, default)  # 安全读取
```

---

## 二、哈希表的核心操作

### 1. 插入和查找（O(1)）
```python
count = {}
count['apple'] = 3       # 插入
count['apple'] += 1      # 更新
print(count['apple'])    # 查找
```

### 2. 安全读取（防止 KeyError）
```python
# ❌ 键不存在会报错
count['banana']          # KeyError!

# ✅ 用 get() 安全读取
count.get('banana')      # None
count.get('banana', 0)   # 0（默认值）
```

### 3. 统计频率
```python
nums = [1, 2, 3, 2, 1]
count = {}
for num in nums:
    count[num] = count.get(num, 0) + 1
# {1: 2, 2: 2, 3: 1}
```

### 4. 判断是否存在
```python
if target - num in hash_map:
    return [hash_map[target-num], i]
```

---

## 三、哈希表 vs 数组 vs 集合

| 结构 | 用途 | 查找复杂度 |
|------|------|-----------|
| 数组 | 按索引存取值 | O(1) 按索引 |
| 集合 | 判断存在、去重 | O(1) |
| **字典** | 键值对映射 | **O(1)** |

## 四、哈希表三大应用场景

### 场景1：两数之和
```python
# 核心思想：找补数
for i, num in enumerate(nums):
    complement = target - num
    if complement in hash_map:
        return [hash_map[complement], i]
    hash_map[num] = i
```

### 场景2：统计频率
```python
# 统计每个字符出现次数
for char in s:
    count[char] = count.get(char, 0) + 1
```

### 场景3：去重
```python
# 集合自动去重
unique_chars = set(s)
```

## 六、注意事项

### 1. 键必须是可哈希的
```python
# ✅ 可哈希：字符串、数字、元组
hash_map['key'] = 1
hash_map[(1, 2)] = 2

# ❌ 不可哈希：列表（会报错）
hash_map[[1, 2]] = 3  # TypeError!
```

### 2. 默认值的选择
```python
# 统计频率：默认 0
count.get(num, 0)

# 找索引：默认 -1
index.get(num, -1)
```

## 七、对比：列表查找 vs 哈希表查找

```python
nums = [1, 2, 3, 4, 5]

# 列表查找：O(n) 逐个比较
if 3 in nums:  # 慢！要检查每个元素

# 哈希表查找：O(1) 直接定位
hash_set = set(nums)
if 3 in hash_set:  # 快！直接计算哈希
```
