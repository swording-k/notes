# 贪心算法

> 一种"局部最优导致全局最优"的算法思想，每步都选当前最优解

---

## 一、贪心基础

### 1.1 什么是贪心

**贪心**：在每一步选择中都采取当前状态下最优的选择，期望通过局部最优达到全局最优。

```
贪心 vs 动态规划：
- 贪心：只看眼前，每步最优 → 可能不是全局最优
- 动态规划：记住之前所有状态 → 保证全局最优
```

### 1.2 贪心能解决的问题

**前提**：问题的最优解可以通过局部最优推导出来

| 问题类型 | 典型题目 | 贪心策略 |
|----------|---------|---------|
| 区间调度 | [[0028-最大子数组和]] | 连续和为负则放弃 |
| 分割问题 | 俄罗斯套娃信封 | 先按一维排序 |
|  Huffman | 合并果子 | 每次选最小两个 |
| 买卖股票 | [[0029-买卖股票最佳时机II]] | 所有上涨都买卖 |

### 1.3 贪心算法模板

```python
def greedy(nums):
    # 1. 排序（如果需要）
    nums.sort()

    # 2. 遍历 + 贪心选择
    result = 0
    for i in range(len(nums)):
        # 贪心选择逻辑
        if condition:
            result += nums[i]

    return result
```

### 1.4 贪心正确性证明思路

```
证明方法：
1. 贪心选择性质：存在最优解以贪心选择开头
2. 最优子结构：当前最优 + 子问题最优 = 全局最优

简单验证：
- 举反例：如果存在反例，贪心就失败
- 归纳法：假设前 k 步都是最优，证第 k+1 步也是
```

## 二、区间问题

### 2.1 最大子数组和（[[0028-最大子数组和]]）

**问题**：找出连续子数组的最大和

**贪心策略**：
- 累加和为负？放弃，从下一个重新开始
- 因为负数和只会让后面的和更小

```python
def maxSubArray(nums):
    max_sum = nums[0]    # 全局最大
    cur_sum = nums[0]    # 当前连续和

    for i in range(1, len(nums)):
        cur_sum = max(nums[i], cur_sum + nums[i])  # 贪心：重新开始 vs 继续累加
        max_sum = max(max_sum, cur_sum)

    return max_sum
```

**图解**：
```
nums = [-2,1,-3,4,-1,2,1,-5,4]

步1: cur=-2, max=-2
步2: cur=max(-3, -2+1)=-1, max=-1   ← 重新开始更好
步3: cur=max(-3, -1-3)=-3, max=-1
步4: cur=max(4, -3+4)=4, max=4      ← 重新开始（之前的和是负数）
步5: cur=max(-1, 4-1)=3, max=4
步6: cur=max(2, 3+2)=5, max=5
步7: cur=max(1, 5+1)=6, max=6
步8: cur=max(-5, 6-5)=1, max=6
步9: cur=max(4, 1+4)=5, max=6

结果：6（[4,-1,2,1]）
```

## 三、买卖股票问题

### 3.1 买卖股票最佳时机 II（[[0029-买卖股票最佳时机II]]）

**问题**：允许多次交易，获取最大利润

**贪心策略**：
- 只要明天会涨，今天就买
- 所有上涨区间都是利润

```python
def maxProfit(prices):
    profit = 0
    for i in range(1, len(prices)):
        if prices[i] > prices[i-1]:
            profit += prices[i] - prices[i-1]
    return profit
```

**图解**：
```
prices = [7,1,5,3,6,4]

      日
      ↓
价格  7   1   5   3   6   4
      │   │   │   │   │   │
      └───┘   └───┘   └───
       亏    涨4    跌2    涨3
      总利润 = (5-1) + (6-3) = 7

等价计算：所有上涨的和 = (1-7)负忽略 + 5-1 + 3-5负忽略 + 6-3 + 4-6负忽略
        = 0 + 4 + 0 + 3 + 0 = 7
```

## 四、分配问题

### 4.1 分发饼干（[[0026-分发饼干]]）

**问题**：每个孩子有胃口 g，每个饼干有尺寸 s，最大化满足的孩子数

**贪心策略**：
- 先用大饼干满足大胃口（或者反过来）
- 排序后贪心分配

```python
def findContentChildren(g, s):
    g.sort()  # 孩子胃口
    s.sort()  # 饼干尺寸
    child = 0  # 孩子索引

    for size in s:  # 遍历每个饼干
        if child < len(g) and size >= g[child]:
            child += 1  # 满足了一个孩子

    return child
```

**图解**：
```
g = [1,2,3], s = [1,1]

排序后：g=[1,2,3], s=[1,1]

饼干1→1: 满足孩子0 (胃口1) ✓
饼干1→1: 满足孩子1 (胃口2) ✗ 不够

结果：1个孩子

如果用大饼干满足大胃口：
s排序后还是[1,1]，一样
```

## 五、摆动序列（[[0027-摆动序列]]）

### 5.1 什么是摆动序列

**摆动序列**：相邻差值正负交替的序列

```
[1,7,4,9,2,5] → 差值 [+6,-3,+5,-7,+3] → 交替 ✓
[1,4,7,2,5] → 差值 [+3,+3,-5,+3] → 有两个连续+ → 不是摆动
```

### 5.2 贪心策略

找到所有"峰"和"谷"，峰谷之间的元素可以删掉

```python
def wiggleMaxLength(nums):
    if len(nums) < 2:
        return len(nums)

    up = down = 1  # 以上升/下降结尾的最长摆动长度

    for i in range(1, len(nums)):
        if nums[i] > nums[i-1]:
            up = down + 1    # 上升段 = 之前下降段 + 1
        elif nums[i] < nums[i-1]:
            down = up + 1    # 下降段 = 之前上升段 + 1

    return max(up, down)
```

## 六、常见错误

### 6.1 误用贪心

```python
# 错误：认为局部最优 = 全局最优
# 例如：取钱问题，每次取最大面额不一定最优

# 例子：需要凑 15 元，币种 [1,6,7]
# 贪心：7 + 7 = 14，不够
# 最优：7 + 6 + 1 + 1 = 15

# 如果用贪心就会失败
```

### 6.2 忘记排序

```python
# 错误：没排序就贪心
for size in s:
    if size >= g[child]:  # g没排序，逻辑就错了

# 正确：先排序
g.sort()
s.sort()
```

## 七、解题框架

```
拿到贪心题，先问：

1. 这题能用贪心吗？
   - 找反例：如果存在反例，贪心失败
   - 证明：局部最优能推导全局最优

2. 贪心策略是什么？
   - 最大/最小优先？
   - 排序后处理？
   - 遇到某个条件就做选择？

3. 代码框架？
   - 排序 + 遍历
   - 双指针
   - 单调栈
```

## 八、贪心 vs 动态规划

| 维度 | 贪心 | 动态规划 |
|------|------|---------|
| 策略 | 每步最优 | 全局最优 |
| 记忆 | 无 | 记住子问题 |
| 适用场景 | 局部最优 = 全局最优 | 需要记录多种状态 |
| 时间复杂度 | 通常 O(n) | 通常 O(n) 或更高 |
| 正确性 | 需要证明 | 一定有最优解 |

**什么时候用贪心？**
- 问题能证明局部最优 → 全局最优
- 或者找不到反例，且贪心明显正确
