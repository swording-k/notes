# Agent（智能体）

> LLM + 工具 + 记忆 + 规划 = Agent

---

## 一、什么是 Agent？

### 一句话定义
```
Agent = 大脑（LLM）+ 手脚（工具）+ 记忆 + 规划
能够感知环境、做决策、执行动作、达成目标的智能系统
```

### Agent vs 普通 ChatGPT

| 维度 | 普通 ChatGPT | Agent |
|------|-------------|-------|
| 能力 | 只能回答 | 能执行操作 |
| 工具 | 无 | 可调用 API、搜索 |
| 记忆 | 无（每次重新开始） | 有（短期+长期） |
| 规划 | 无 | 有（拆解任务） |
| 例子 | 回答问题 | 真正完成任务 |

## 二、Agent 的四大组件

```
                    ┌─────────────────────────────────────┐
                    │              用户输入                 │
                    └──────────────────┬──────────────────┘
                                       ↓
                    ┌─────────────────────────────────────┐
                    │          LLM（大脑）                  │
                    │    理解意图 → 做决策 → 生成回复       │
                    └──────────────────┬──────────────────┘
                                       ↓
              ┌────────────────────────┼────────────────────────┐
              ↓                        ↓                        ↓
    ┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
    │   工具 (Tools)   │      │   记忆 (Memory)  │      │   规划 (Planner)│
    │                 │      │                 │      │                 │
    │ 搜索/API/计算   │      │ 短期+长期记忆    │      │ 任务拆解        │
    └─────────────────┘      └─────────────────┘      └─────────────────┘
```

### 1. LLM（大脑）
```python
# 核心能力：理解意图、做决策
llm = ChatOpenAI(model="gpt-4")
response = llm.invoke("帮我查北京天气")
```

### 2. Tools（工具）
```python
# 工具让 Agent 有"手脚"
tools = [
    search_tool,      # 搜索
    calculator_tool,  # 计算
    api_tool,         # API 调用
]
```

### 3. Memory（记忆）
```python
# 短期记忆：当前对话上下文
# 长期记忆：向量数据库存储的历史
memory = ConversationBufferMemory()
```

### 4. Planner（规划）
```python
# 把大任务拆成小步骤
plan = """
1. 搜索北京天气
2. 根据温度建议穿着
3. 输出回答
"""
```

## 三、Agent 执行策略

### 1. ReAct（推理 + 行动）

#### 核心思想
```
思考 → 行动 → 观察 → 循环
```

#### 执行流程
```
用户：帮我查北京天气，然后告诉我穿什么

Step 1: 思考 → "我需要先查天气"
Step 2: 行动 → 调用天气API工具
Step 3: 观察 → 得到北京25°C，晴天
Step 4: 思考 → "温度适中，建议轻薄衣服"
Step 5: 行动 → 输出回答

图示：
        ┌───────┐
        ↓       ↓
    思考    行动 → 工具调用
        ↑       ↓
        └── 观察 ←─┘
```

#### 适用场景
- 环境会变化、需要根据结果调整
- 需要多步推理
- 搜索 + 计算组合任务

---

### 2. CoT（Chain of Thought）

#### 核心思想
```
先把计划想清楚，再执行
```

#### 适用场景
- 任务可规划、步骤明确
- 复杂数学推理
- 代码生成

```python
# CoT 示例
thought = """
1. 首先分析问题...
2. 然后拆解步骤...
3. 最后执行...
"""
```

---

### 3. 其他策略

| 策略 | 核心思想 | 适用场景 |
|------|---------|---------|
| ReAct | 思考→行动→观察 | 动态环境、搜索 |
| CoT | 先想后做 | 明确任务、推理 |
| Plan-and-Execute | 规划后再执行 | 复杂多步骤 |
| Reflexion | 行动后反思改进 | 试错场景 |

## 四、Agent 的实际例子

### 例子1：Claude Code

```python
# Claude Code 本质是一个 Agent
# 架构：
用户输入 → LLM(Claude) → 规划 → 执行工具
                              ├── Read（读文件）
                              ├── Write（写文件）
                              ├── Bash（执行命令）
                              ↓
                         观察结果 → 调整 → 继续
```

**为什么 Claude Code 比 ChatGPT 更强？**
```
ChatGPT：只能回答问题
Claude Code：能读文件、修改代码、执行命令
→ 真正帮你完成编程任务
```

### 例子2：客服机器人

```
用户：帮我查订单状态

Agent 处理：
1. 理解意图：查订单
2. 调用工具：订单系统 API
3. 获取数据：订单状态
4. 组织回答：返回给用户
```

## 六、面试必备回答

### Q: 什么是 Agent？
```
Agent 是能够感知环境、做决策、执行动作的智能系统。

核心组成：
1. LLM：大脑，理解意图、做决策
2. 工具：手脚，执行搜索/API/计算
3. 记忆：保存对话历史
4. 规划：拆解大任务为小步骤

和普通 ChatGPT 的区别：
Agent 不只能回答问题，还能真正完成任务。
```

### Q: ReAct 执行策略？
```
ReAct = Reasoning + Acting（推理 + 行动）

流程：思考 → 行动 → 观察 → 循环

例如：查天气穿衣
1. 思考："需要先查天气"
2. 行动：调用天气 API
3. 观察："得到 25°C"
4. 思考："温度适中"
5. 行动：输出建议
```

### Q: 能举一个 Agent 的实际例子吗？
```
Claude Code 是一个典型的 Agent 应用。

技术架构：
用户给编程任务 → LLM 理解意图 → 规划执行步骤
→ 通过工具执行（读文件、写文件、执行命令）
→ 观察结果 → 调整下一步

和 ChatGPT 的区别：
能操作文件系统、终端，真正完成编程任务。
```
