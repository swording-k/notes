# RAG（检索增强生成）

> 让大模型"读懂"自己的知识库

---

## 一、什么是 RAG？

### 全称
```
RAG = Retrieval Augmented Generation（检索增强生成）
```

### 一句话解释
```
把知识库准备好 → 用户问问题 → 检索相关知识 → 注入上下文 → 让模型基于知识回答
```

### 为什么需要 RAG？
```
LLM 的问题：
1. 知识有截止日期（不知道最新信息）
2. 可能产生幻觉（一本正经胡说八道）
3. 没有企业私有知识

RAG 解决方案：
→ 让模型"看"真实文档再回答
→ 准确、可信、可追溯
```

## 二、RAG 工作流程

```
                    文档处理阶段
    ┌─────────────────────────────────────────┐
    │                                         │
    │  原始文档 → 分块 → Embedding → 向量库   │
    │                                         │
    └─────────────────────────────────────────┘
                        ↓
                    用户查询
    ┌─────────────────────────────────────────┐
    │                                         │
    │  用户问题 → Embedding → 相似度检索 →    │
    │  检索 Top-K → 构建上下文 → 注入 Prompt  │
    │                                         │
    └─────────────────────────────────────────┘
                        ↓
                    LLM 生成
    ┌─────────────────────────────────────────┐
    │                                         │
    │  Prompt + 上下文 → LLM → 生成回答       │
    │                                         │
    └─────────────────────────────────────────┘
```

### 详细步骤

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 文档加载 | 读取 PDF/Word/TXT 等 |
| 2 | 文本分块 | 切成 300-500 字的小块 |
| 3 | 向量化 | 用 Embedding 模型转成向量 |
| 4 | 存入向量库 | Milvus/Pinecone/Chroma |
| 5 | 用户问题向量化 | 把问题也转成向量 |
| 6 | 相似度检索 | Top-K 相关文档块 |
| 7 | 构建 Prompt | 把检索结果注入上下文 |
| 8 | LLM 生成 | 基于知识回答 |

## 四、关键概念详解

### 1. Embedding（向量化）
```python
# 文本 → 向量（数字数组）
text = "什么是 RAG"
vector = embedding_model.encode(text)
# vector = [0.123, -0.456, 0.789, ...]  # 1536 维度
```

**为什么用向量？**
```
文字匹配：只能匹配完全相同的词
向量匹配：语义相似也能量化
"苹果水果" ≈ "水果苹果"  → 向量相似度高 ✅
"苹果手机" ≈ "iPhone"   → 向量相似度高 ✅
```

### 2. 文本分块（Chunking）
```python
# 常用方法：按字数分段
text = "这是一段很长的文档内容..."
chunks = []
for i in range(0, len(text), 300):  # 每300字一块
    chunks.append(text[i:i+300])
```

**分块策略选择：**
| 策略 | 适用场景 | 块大小 |
|------|---------|--------|
| 固定字数 | 通用 | 300-500 字 |
| 句子分割 | 对话类 | 按句号分割 |
| 语义分割 | 复杂文档 | 按段落分割 |

### 3. 向量检索 vs 关键词检索

| 维度 | 传统检索（BM25） | 向量检索 |
|------|----------------|---------|
| 原理 | 关键词匹配 | 语义相似度 |
| 效果 | 查不准同义词 | 理解语义 |
| 速度 | 快 | 相对慢 |
| 例子 | "手机" 匹配不到 "phone" | "手机" 和 "phone" 相似度高 |

## 五、RAG 进阶技巧

### 1. Hybrid Search（混合检索）
```python
# 结合关键词 + 向量
results = bm25_search(query) + vector_search(query)
# 取两者的交集或并集
```

### 2. Rerank（重排序）
```python
# 第一轮：快速向量检索（Top-100）
# 第二轮：精排模型重排序（Top-10）
```

### 3. Query 改写
```python
# 用户说 "它" → 改成完整问题
query = "它的原理是什么"  # ❌ 不知道"它"是什么
query = "RAG的原理是什么"  # ✅ 完整问题
```

## 六、RAG vs Fine-tuning

| 维度 | RAG | Fine-tuning |
|------|-----|-------------|
| 成本 | 低（不用训练） | 高（需要训练） |
| 更新速度 | 快（换文档就行） | 慢（重新训练） |
| 准确性 | 高（基于真实文档） | 取决于训练数据 |
| 适用场景 | 知识库问答 | 垂直领域优化 |

## 八、面试必备回答

### Q: 什么是 RAG？
```
RAG（检索增强生成）是一种让大模型"看"知识库再回答的技术。

工作流程：
1. 文档分块 → 向量化 → 存入向量数据库
2. 用户问题 → 向量化 → 相似度检索
3. 检索结果 → 注入 Prompt → LLM 生成回答

解决的问题：
- LLM 知识截止问题
- 减少幻觉，提高准确性
- 支持企业私有知识库
```

### Q: RAG 的核心组件？
```
1. 文档加载器：读取 PDF/Word 等
2. 文本分块：切成小块（一般 300-500 字）
3. Embedding 模型：将文本转为向量
4. 向量数据库：存储和检索向量（Milvus/Pinecone/Chroma）
5. LLM：根据上下文生成回答
```
