## 一、高频概念题（必须背熟）

### 1. RAG 相关

**Q: 什么是RAG？为什么需要RAG？**

```
RAG = Retrieval Augmented Generation（检索增强生成）

为什么需要：
1. LLM知识有截止日期，不知道最新信息
2. LLM可能产生幻觉（胡说八道）
3. 企业有私有知识库，需要接入LLM
4. 提高回答的准确性和可信度

工作流程（背下来）：
文档 → 分块 → 向量化 → 存入向量数据库
                    ↓
用户问题 → 向量化 → 相似度检索 → 上下文注入 → LLM生成 → 回答
```

**Q: RAG的核心组件有哪些？**

| 组件          | 作用                              |
| ----------- | ------------------------------- |
| 文档加载器       | 读取PDF/Word/TXT等                 |
| 文本分块        | 将长文档切成小块（一般300-500字）            |
| Embedding模型 | 将文本转为向量（用余弦相似度匹配）               |
| 向量数据库       | 存储和检索向量（Milvus/Pinecone/Chroma） |
| LLM         | 根据上下文生成回答                       |

**Q: 向量检索和传统关键词检索的区别？**

| 维度 | 传统检索（BM25） | 向量检索  |
| -- | ---------- | ----- |
| 原理 | 关键词匹配      | 语义相似度 |
| 效果 | 查不准同义词     | 理解语义  |
| 速度 | 快          | 相对慢   |

### 2. Agent 相关

**Q: 什么是Agent？由哪些组件构成？**

```
Agent = LLM（大脑） + 工具（手和脚） + 记忆 + 规划

核心组件：
1. LLM：理解用户意图，做决策
2. 工具调用：搜索、计算、API调用
3. 记忆：短期记忆（上下文）+ 长期记忆（向量库）
4. 规划：将大任务拆解为小步骤
```

**Q: Agent有哪些执行策略？ReAct是唯一的吗？**

```
不是！ReAct只是其中一种执行策略。

常见的Agent执行策略：
├── ReAct（Reasoning + Acting）：思考→行动→观察→循环（适合动态环境）
├── CoT（Chain of Thought）：先把计划想清楚，再执行
└── Action Branching：同时探索多条执行路径

ReAct适合：环境会变化、需要根据结果调整的情况
CoT适合：任务可规划、步骤明确的情况
```

**Q: ReAct的执行例子（能画出来）？**

```
用户：帮我查北京天气，然后告诉我穿什么

步骤1：思考 → "我需要先查天气，再根据温度建议穿着"
步骤2：行动 → 调用天气API工具
步骤3：观察 → 得到北京25°C，晴天
步骤4：思考 → "温度适中，建议穿轻薄衣服"
步骤5：行动 → 输出回答
```

**Q: 什么是Claude Code？它的技术原理是什么？**

```
Claude Code是一个典型的AI Agent应用，运行在终端/IDE里的AI编程助手。

技术架构（能画出来）：
┌─────────────────────────────────────┐
│            用户输入                  │
│      "帮我重构这个函数"              │
└─────────────┬───────────────────────┘
              ↓
┌─────────────────────────────────────┐
│          LLM (Claude)                │
│      理解意图，做决策                │
└─────────────┬───────────────────────┘
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
┌────────┐┌────────┐┌────────┐
│ Read  ││ Write  ││ Bash   │
│ 读文件 ││ 写文件 ││执行命令│
└────────┘└────────┘└────────┘

和普通ChatGPT的区别：
- ChatGPT：只能回答，不能操作
- Claude Code：能读文件、执行命令，真正帮用户干活
```

**面试时能说的例子**：

```
"Claude Code 本质上是一个 'LLM + 工具 + 规划' 的 Agent 系统。
用户给编程任务 → LLM理解意图 → 规划执行步骤 → 通过工具执行
（读文件、写文件、执行命令）→ 根据结果调整下一步。

它和普通 ChatGPT 的区别是，它接入了文件系统、终端等工具，
能够真正完成编程任务，而不只是回答问题。"
```

### 3. Prompt Engineering

**Q: Prompt工程的基本原则？**

```
1. 清晰具体的指令（别让模型猜）
2. 给示例（Few-shot）
3. 分解复杂任务（Chain of Thought）
4. 指定输出格式
5. 限制输出长度（max_tokens）
```

**Q: 什么是Few-shot和Zero-shot？**

| 模式        | 说明        | 例子                      |
| --------- | --------- | ----------------------- |
| Zero-shot | 什么都不给，直接问 | "翻译：hello"              |
| Few-shot  | 给几个示例     | "你好=hello，好=good，翻译：世界" |

## 二、LLM基础知识

**Q: LLM有哪些局限性？**

1. **幻觉**：一本正经地胡说八道
2. **知识截止**：不知道最新信息
3. **上下文限制**：有最大token限制（如4K、128K）
4. **推理能力**：复杂推理不如人类
5. **多模态**：默认只处理文本

**Q: 如何缓解LLM的幻觉问题？**

```
1. RAG：让模型参考真实文档
2. CoT：让模型一步步推理
3. 提示工程：要求模型"不确定时说不知道"
4. 人工审核：重要场景人工复核
```

**Q: GPT-4和GPT-3.5的区别？**

```
GPT-4:
- 更大参数规模
- 多模态（能看图）
- 更强的推理能力
- 更长的上下文（128K vs 4K）

GPT-3.5:
- 更快更便宜
- 适合简单任务
```

## 三、框架知识（能说出来就行）

### Q: 什么是LangChain？它和代码模板有什么区别？

**一句话解释**：
> LangChain = **AI应用开发的工具箱**（不是模板，是一堆现成的代码工具）

**和模板的区别**：

| 区别 | 代码模板 | 框架 |
| ---- | -------- | ---- |
| 模板 | 填参数，生成代码 | 拼积木，组装功能 |
| 框架 | 一次性使用 | 可以反复调用 |
| 比如 | 你填表单生成网站 | Flask直接跑网站 |

**类比理解**：
```
传统开发:
你要发HTTP请求 → 手写Socket/协议 → 很麻烦

用 requests 库:
你要发HTTP请求 → requests.get() → 简单！

AI开发也一样:
你要搭AI应用 → 手写API调用代码 → 很麻烦

用 LangChain:
你要搭AI应用 → 调用现成模块 → 简单！
```

---

### LangChain 具体解决什么问题？

**问题1：每次调LLM要写一堆重复代码**
```python
# 不用框架
headers = {"Authorization": f"Bearer {api_key}"}
data = {"model": "gpt-4", "messages": [...]}
response = requests.post(url, headers=headers, json=data)

# 用LangChain
llm = ChatOpenAI(model="gpt-4")
result = llm.invoke("你好")
```

**问题2：LLM不知道怎么调用外部工具**
```python
# 不用框架：自己处理工具调用逻辑（复杂）

# 用LangChain：
tools = [search_tool, calculator_tool]
agent = create_agent(llm, tools)
agent.run("查天气再计算10%税")
# LLM自动决定用哪个工具
```

**问题3：LLM不记得之前说过的话**
```python
# 不用框架：自己存对话历史、自己拼接prompt

# 用LangChain：
memory = ConversationBufferMemory()
memory.save_context({"input": "我叫张三"}, {"output": "你好张三"})
# 自动管理记忆，LLM自然"记得"
```

### LangChain 5个核心概念（必须记住）

| 概念 | 通俗理解 | 生活中的例子 |
| ---- | -------- | ------------ |
| **PromptTemplate** | 提示词模板 | 填表格的模板（姓名____日期____） |
| **Chain** | 流水线 | 工厂流水线（原料→加工→成品） |
| **Tool** | 工具 | 瑞士军刀（刀、剪、开瓶器...） |
| **Agent** | 智能体 | 机器人（会思考、会行动） |
| **Memory** | 记忆 | 聊天记录（自动保存） |

---

### LlamaIndex 和 LangChain 的区别？

```
LangChain：全流程框架，更通用（什么都能做）
LlamaIndex：专注数据接入（RAG），更专业

实际选择：
- 做RAG系统 → 优先 LlamaIndex
- 做复杂Agent → 优先 LangChain
- 也可以结合使用
```

**Q: 什么是LangChain？**

```
"LangChain是一个AI应用开发框架，类似于Web开发里的Flask。

它的作用是把AI应用开发的常见操作封装成现成的模块，
比如：
- PromptTemplate 管理提示词
- Chain 把多个步骤串成流水线
- Tool 让AI能调用外部API
- Memory 自动管理对话历史

我理解它的核心价值是：不用从零写代码，直接拼积木。"
```
